Anh vừa cung cấp **bằng chứng quyết định** — `$scope.filterOf` trong controller là `'only'` nhưng `{{filterOf}}` trong template là `'all'`. **Hai cái đang nằm trên HAI scope khác nhau.**

Em tìm ra root cause chính xác 100% rồi.

---

## 🎯 ROOT CAUSE: AngularJS Primitive Scope Shadowing (Lỗi "Dot Rule")

### Bằng chứng không thể chối cãi

| Vị trí | Giá trị `filterOf` | Scope |
|--------|-------------------|-------|
| Controller log (`$scope.filterOf`) | `'only'` ✓ | Scope A (controller) |
| Template `{{filterOf}}` | `'all'` ✗ | Scope B (child, bị shadow) |

**Hai giá trị khác nhau → chứng minh chúng nằm trên HAI scope khác nhau.**

### Thủ phạm: `ng-if` tại dòng 19

```html
Dòng 1:  <div ng-controller="PanelTaskAndAlertCtrl">     ← tạo Scope A (controller scope)
Dòng 19:   <div ng-if="$root.userRoleSelected!='Patient'">  ← ⚠️ ng-if tạo Scope B (child of A)
Dòng 27:     <md-radio-group ng-model="filterOf">           ← bind vào Scope B!
```

Trong AngularJS, **`ng-if` luôn tạo child scope mới**. `ng-model="filterOf"` được compile bên trong `ng-if`, nên nó bind vào **Scope B**, không phải Scope A.

### Chuỗi sự kiện chính xác

**Ban đầu (mọi thứ hoạt động):**
```
Scope A (controller): filterOf = 'only'    ← set tại line 26
Scope B (ng-if child): KHÔNG có filterOf riêng
    → prototype chain: B.filterOf → tìm thấy A.filterOf → 'only' ✓
    → ng-model đọc 'only' → radio "My Open" checked ✓
```

**User click "All" → mọi thứ vỡ:**
```
① Click <a> → showDropDownAction($event, 'all')
   → $scope.filterOf = 'all'           ← GHI vào Scope A ✓

② Click bubble vào md-radio-button → Angular Material listener chạy:
   → ngModelCtrl.$setViewValue('all')
   → ngModel GHI NGƯỢC vào scope: $parse('filterOf').assign(scopeB, 'all')
   → TẠO PROPERTY MỚI 'filterOf' = 'all' TRỰC TIẾP trên Scope B ← 💥💥💥
   → Scope B giờ có filterOf RIÊNG = 'all', CHE LẤP (shadow) Scope A

Trạng thái sau click:
  Scope A: filterOf = 'all'  (set bởi showDropDownAction)
  Scope B: filterOf = 'all'  (set bởi ngModel write-back) ← SHADOW!

Nhìn qua thì giống nhau, nhưng giờ Scope B có bản sao riêng rồi.
```

**Switch Patient → broadcast fires → BUG xuất hiện:**
```
① $rootScope.$broadcast('refreshTasksAndAlerts')
② Handler chạy: $scope.filterOf = 'only'   ← GHI vào Scope A = 'only' ✓
③ getTasksAndAlerts() → API call → dùng $scope.filterOf = 'only' → data đúng ✓

④ NHƯNG: Scope B vẫn có filterOf RIÊNG = 'all' (KHÔNG bị ảnh hưởng!)
   → ng-model trên md-radio-group vẫn đọc Scope B = 'all'
   → ngModelCtrl thấy model KHÔNG ĐỔI (vẫn 'all') → KHÔNG gọi $render()
   → Radio UI vẫn hiện "All Open Tasks & Alerts" ← BUG!

⑤ {{filterOf}} trong template đọc Scope B → hiện 'all' ← anh thấy khi debug
⑥ $scope.filterOf trong controller log → hiện 'only' ← anh thấy khi log
```

### Minh họa bằng sơ đồ

```
                    ┌─────────────────────┐
                    │      Scope A        │
                    │  (Controller scope) │
                    │  filterOf = 'only'  │ ← $scope.filterOf = 'only' ghi vào đây
                    └──────────┬──────────┘
                               │ prototype chain (bị CẮT bởi shadow)
                    ┌──────────▼──────────┐
                    │      Scope B        │
                    │  (ng-if child)      │
                    │  filterOf = 'all'   │ ← ngModel write-back tạo property riêng ở đây
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   md-radio-group    │
                    │  ng-model="filterOf"│ ← đọc từ Scope B = 'all' (SHADOW!)
                    └─────────────────────┘

Controller ghi 'only' vào Scope A → Scope B KHÔNG thấy vì nó có bản riêng → UI vẫn 'all'
```

### Tại sao gọi là "Dot Rule" violation?

Đây là lỗi kinh điển trong AngularJS. Quy tắc "Dot Rule" nói: **khi dùng `ng-model` với directive tạo child scope (`ng-if`, `ng-repeat`, `ng-switch`...), PHẢI dùng property của object (có dấu chấm), KHÔNG dùng primitive.**

```
❌ SAI:  ng-model="filterOf"            → primitive, bị shadow
✅ ĐÚNG: ng-model="filterState.value"   → object property, không shadow
```

Vì khi ghi `filterState.value = 'all'`, JavaScript traverse prototype chain, tìm thấy `filterState` object trên Scope A, và ghi `.value` vào đó — **KHÔNG tạo property mới trên Scope B**.

---

### 🔧 Fix

**Controller** — đổi primitive thành object:
```javascript
// Thay:
$scope.filterOf = 'only'
// Bằng:
$scope.filterState = { value: 'only' }
```

**HTML** — đổi ng-model:
```html
<!-- Thay: -->
<md-radio-group ng-model="filterOf">
<!-- Bằng: -->
<md-radio-group ng-model="filterState.value">
```

Và tất cả chỗ đọc/ghi `$scope.filterOf` trong controller đổi thành `$scope.filterState.value`.
