**Làm theo Solution B từng bước**

1. Sửa template radio để chỉ có **1 nguồn state** (`ng-model` + `ng-change`) trong [open-tasks-and-alert.html](/d:/Sources/pteverywhere/Client/app/views/patientDashboard/profile-partial/open-tasks-and-alert.html).  
Thay block cũ bằng:

```html
<md-radio-group ng-model="filterOf" ng-change="onFilterChanged()" aria-labelledby="1500-patient-cond-other">
  <md-radio-button value="only" class="md-primary m-0 dropdown-item">{{::('Profile.MyAlerts' | translate)}} 5</md-radio-button>
  <md-radio-button value="all" class="md-primary m-0 dropdown-item"
    ng-if="$root.userRoleSelected=='Admin' || $root.userRoleSelected=='SystemAdmin' || ($root.userRoleSelected=='Therapist' && $root.userEnableAdminRight)">
    {{::('Profile.AllAlerts' | translate)}} 6
  </md-radio-button>
</md-radio-group>
```

2. Trong [panelTaskAndAlertController.js](/d:/Sources/pteverywhere/Client/app/scripts/controllers/patientDashboard/panelTaskAndAlertController.js), tạo hàm dùng chung để reset + reload:
```javascript
function applyFilterAndReload (filterValue, reason) {
  if (filterValue) { $scope.filterOf = filterValue }
  $scope.isLoadingTasksAndAlerts = true
  $scope.totalTasks = 0
  $scope.tasksAndAlerts = []
  $scope.pageSelected = 1
  getTasksAndAlerts()
}
```

3. Thêm handler cho `ng-change`:
```javascript
$scope.onFilterChanged = function () {
  applyFilterAndReload($scope.filterOf, 'radio-change')
}
```

4. Giữ `showDropDownAction` để backward-compatible nhưng đổi nó sang gọi chung:
```javascript
$scope.showDropDownAction = function ($event, filterOf) {
  $event && $event.preventDefault && $event.preventDefault()
  applyFilterAndReload(filterOf, 'dropdown-click')
}
```

5. Thêm watcher khi đổi patient id để luôn reset về `only`:
```javascript
var unwatchPatientId = $scope.$watch(function () {
  var patient = $scope.parentScope && ($scope.parentScope.patientDataView || $scope.parentScope.profile)
  return patient && patient._id ? String(patient._id) : null
}, function (newId, oldId) {
  if (!newId) { return }
  if (!oldId) { return } // lần init đầu
  if (newId !== oldId) {
    applyFilterAndReload('only', 'patient-switch')
  }
})
```
Và trong `$destroy` nhớ `unwatchPatientId && unwatchPatientId()`.

6. Thêm guard chống stale response trong `fetchTasksAndAlerts()`:
```javascript
$scope._latestFetchRequestId = $scope._latestFetchRequestId || 0
var requestId = ++$scope._latestFetchRequestId
var currentPatientId = patientDataView && patientDataView._id ? String(patientDataView._id) : null
```
Trong `.then(...)`:
```javascript
if (requestId !== $scope._latestFetchRequestId) { return }
var latestPatient = $scope.parentScope && ($scope.parentScope.patientDataView || $scope.parentScope.profile)
if (latestPatient && String(latestPatient._id) !== currentPatientId) { return }
```
Trong `.finally(...)`:
```javascript
if (requestId === $scope._latestFetchRequestId) {
  $scope.isLoadingTasksAndAlerts = false
}
```

7. Đổi event refresh sang hàm chung:
```javascript
$scope.filterOf = 'only'; getTasksAndAlerts();
```
thành:
```javascript
applyFilterAndReload('only', 'refresh-event')
```
