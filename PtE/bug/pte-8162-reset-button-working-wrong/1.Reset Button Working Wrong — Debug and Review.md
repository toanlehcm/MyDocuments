# PTE-8162: Reset Button Working Wrong — Debug & Review

## 📌 Ticket Info

**PTE-8162** - [RCM → Invoice] Reset button working wrong

**Steps to reproduce:**
1. Click view Quick View
2. Remove column DOS and click save
3. Add X column that is not DOS
4. Click reset button

**Observed:** Show column DOS
**Expected:** Not show column DOS

---

## #2. 🔍 Debug: Root Cause Analysis

### Root Cause chính: ✅ Anh ĐÚNG — `view.DisplayColumns = res.DisplayColumns` là key fix

#### Giải thích cơ chế kỹ thuật

**Vấn đề nằm ở `oldView` — biến closure lưu trạng thái "last saved" của view.**

Call chain khi Reset:

```
_resetView()
  → _processSelectQuickView(oldView.view)       // L3612
    → _renderViewToPanelFilter(oldView.view)     // L3333
      → oldView.view = _.cloneDeep(view)         // L3251 ← GHI ĐÈ oldView!
      → render columns từ view.DisplayColumns    // L3273-3289
```

**Flow bug (TRƯỚC fix):**

```
Bước 1: User click "Today" view → oldView.view.DisplayColumns = ["InvoiceID", "DOS", ...]
                                   quickViewList[0].DisplayColumns = ["InvoiceID", "DOS", ...]

Bước 2: User remove DOS, click Save
        → API gọi _upsertCustomView() 
        → Server trả về res = { DisplayColumns: ["InvoiceID", ...] } (không có DOS)
        
        ❌ TRƯỚC FIX: Code chỉ update view.Name = res.Name
        → quickViewList[0].DisplayColumns VẪN = ["InvoiceID", "DOS", ...] ← STALE!

Bước 3: User add column X
        → $scope.searchTerms.columns = ["InvoiceID", "ColumnX", ...]
        → searchTerms.isEdited = true

Bước 4: User click Reset
        → _processSelectQuickView(oldView.view)
        → oldView.view là bản CLONE của quickViewList entry (từ lần render trước)
        → oldView.view.DisplayColumns = ["InvoiceID", "DOS", ...] ← VẪN CÒN DOS!
        → _renderViewToPanelFilter render lại columns → HIỆN DOS
```

**Flow đúng (SAU fix):**

```
Bước 2 (SAU FIX):
        → view.Filters = res.Filters;             // sync Filters
        → view.DisplayColumns = res.DisplayColumns; // sync DisplayColumns  
        → quickViewList[0].DisplayColumns = ["InvoiceID", ...] ← ĐÚNG, không có DOS

Bước 4: User click Reset
        → _processSelectQuickView(oldView.view) 
        → nhưng oldView.view sẽ được rebuild...
```

> [!WARNING]
> **KHOAN!** Có 1 vấn đề tiềm ẩn. Xem thêm bên dưới.

### Root Cause thứ 2: `oldView` reference chain

Em trace kỹ hơn. `_processSelectQuickView` gọi `_renderViewToPanelFilter` → `_renderViewToPanelFilter` LUÔN set:

```javascript
// _renderViewToPanelFilter (L3250-3251)
oldView.Name = view.Name;
oldView.view = _.cloneDeep(view);  // ← deep clone
```

**Vậy `oldView.view` bị set lại mỗi lần `_renderViewToPanelFilter` chạy.**

Khi chạy `_saveView` **TRƯỚC fix**:

```
Save thành công:
  → view.Name = res.Name           // ← chỉ update Name
  → (KHÔNG update DisplayColumns, Filters)
  → _calculateWidthOfQuickView()
  
  → NHƯNG KHÔNG gọi lại _renderViewToPanelFilter
  → oldView.view vẫn giữ DisplayColumns CŨ (có DOS)
```

**Đây mới là ROOT CAUSE thật sự:** Sau khi Save, `oldView.view` KHÔNG được update với data mới từ server. Khi Reset, nó dùng `oldView.view` cũ (có DOS) để render lại panel.

**Fix `view.DisplayColumns = res.DisplayColumns` giải quyết được vì:**

1. `view` chính là reference trong `quickViewList` → update DisplayColumns sync
2. Sau đó `.finally(() => _applyView(false))` gọi lại → trigger re-render
3. **NHƯNG** `oldView.view` vẫn chưa được update trực tiếp...

### 🔴 Root Cause thứ 3 (ẨN): `oldView` vẫn stale sau `.finally`

Xem kỹ flow sau khi fix:

```
_saveView()
  → res trả về từ server
  → view.DisplayColumns = res.DisplayColumns  // ← update quickViewList entry
  → .finally(() => _applyView(false))
     → _applyView gọi API search → render table
     → NHƯNG _applyView KHÔNG gọi _renderViewToPanelFilter
     → oldView.view VẪN stale!
```

**Vậy sau Save, nếu user KHÔNG thay đổi gì thêm và click Reset → sẽ dùng oldView cũ?**

**KHÔNG.** Vì:
1. Sau Save → `isEdited = false` (L3655)
2. Reset button **CHỈ HIỆN** khi `searchTerms.isEdited = true` (HTML L362)
3. User phải edit gì đó (add column X) → `isEdited = true` → Reset mới hiện
4. Khi user edit → **watcher** detect changes so sánh với `oldView.columns`

**Nhưng `oldView.view.DisplayColumns` vẫn stale** — nó sẽ chứa bản cũ cho đến khi user **chọn lại** view (trigger `_processSelectQuickView` → `_renderViewToPanelFilter`).

**Trường hợp vẫn bug:**
```
1. Đang ở view "Today" (có DOS)
2. Remove DOS → Save ← Fix update quickViewList
3. Add column X ← isEdited = true, Reset hiện
4. Click Reset
   → _resetView → _processSelectQuickView(oldView.view) 
   → oldView.view = snapshot từ TRƯỚC save (còn DOS!)
   → _renderViewToPanelFilter(oldView.view) → RENDER DOS LẠI!
```

> [!CAUTION]
> **Fix hiện tại CHƯA ĐỦ nếu `_resetView` dùng `oldView.view` mà `oldView.view` chưa được sync sau save.**
>
> **So sánh với ATP:** ATP `_resetView` gọi `_getView($scope.searchTerms.viewId)` — **fetch lại từ server** (L1188), không dùng `oldView` cũ. Đây là cách an toàn hơn.

### Tổng kết Root Cause

| # | Root Cause | Mức độ | Fix hiện tại giải quyết? |
|---|-----------|--------|--------------------------|
| 1 | `quickViewList[].DisplayColumns` stale sau save | 🔴 Chính | ✅ Fix bằng `view.DisplayColumns = res.DisplayColumns` |
| 2 | `oldView.view` không được sync sau save | 🔴 Tiềm ẩn | ⚠️ CHƯA — oldView.view vẫn stale |
| 3 | Thiếu `_invalidToSentRequest()` guard | 🟡 Phụ | ✅ Đã thêm |

### Đề xuất fix triệt để

**Option A (Đơn giản nhất):** Sau khi update `view`, gọi `_renderViewToPanelFilter(view)` để sync `oldView`:

```javascript
if (view) {
  view.Name = res.Name;
  view.Filters = res.Filters;
  view.DisplayColumns = res.DisplayColumns;
  $scope.quickView.selectedQuickView = res._id;
  _renderViewToPanelFilter(view);  // ← SYNC oldView
}
```

**Option B (Giống ATP — An toàn nhất):** Đổi `_resetView` để gọi API `_getView()` fetch data mới từ server thay vì dùng `oldView` cũ.

### Code hiện tại (L3644-3689)

```javascript
function _saveView() {
  if (_isInvalidToSave() || _invalidToSentRequest() || !$scope.permissionAdmin) {
    return;
  }
  const model = _convertToModel();
  _upsertCustomView(model).then(
    (res) => {
      if (res.errorCode && res.errorCode === "VIEW_DELETED") {
        bootbox.alert($translate.instant("InvoiceTab.WarningCannotSaveViewHasBeenDeleted"));
        _loadDefaultViewOfSystem();
      } else if (res) {
        $scope.searchTerms.isEdited = false;
        commonService.showToast(
          $translate.instant("InvoiceTab.NoticeSuccessSaveView"),
          5000, null, null, false, null, null, null
        );
        if (model._id) {
          const view = $scope.quickView.quickViewList.find((i) => i._id === res._id);
          if (view) {
            view.Name = res.Name;
            view.Filters = res.Filters;
            view.DisplayColumns = res.DisplayColumns;
            $scope.quickView.selectedQuickView = res._id;
          }
          _calculateWidthOfQuickView();
        } else {
          res.selected = true;
          _syncInfoAfterSave(res);
        }
      }
    },
    (error) => {
      console.log();
    }
  )
  .finally(() => {
    _applyView(false);
  });
}
```
