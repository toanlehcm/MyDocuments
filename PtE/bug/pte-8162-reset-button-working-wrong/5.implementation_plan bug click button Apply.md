# 🔍 PTE-8162: Root Cause Analysis — Reset + Apply Bug (Invoice vs Claims)

## Bug Scenario

1. Remove column **DOS** từ list panel bên phải
2. Không save, click **Apply** → DOS mất trong table và list panel
3. Mở panel lại, click **Reset** → DOS có trong list panel nhưng **KHÔNG có trong table**
4. Click **Apply** lại → DOS **vẫn không có** trong table

## Mong muốn

Sau khi click **Reset**, column DOS nên xuất hiện ở **cả list panel phải VÀ table trái**.

---

## So sánh luồng Invoice vs Claims

### Cả 2 tab đều có flow giống nhau

```
Reset → _getView(viewId) → _processSelectQuickView() → _applyView(false)
Apply → _applyView(true) → gọi API lấy data → set headerDisplayColumns = searchTerms.columns
```

### Điểm giống nhau ✅

| Hành vi | Invoice | Claims |
|---------|---------|--------|
| `_resetView()` | Gọi `_getView()` → `_processSelectQuickView()` → `_applyView(false)` | **Giống hệt** |
| `_processSelectQuickView()` | Reset `allColumns.selected = false` → `_renderViewToPanelFilter()` | **Giống hệt** |
| `_renderViewToPanelFilter()` | Set `searchTerms.columns` từ `view.DisplayColumns` + mark `allColumns.selected` | **Giống hệt** |

### 🔴 Điểm KHÁC BIỆT QUAN TRỌNG — `_applyView()`

> [!CAUTION]
> **Root cause nằm ở đây**: Khi `_applyView()` kiểm tra `isEdited` flag, nó quyết định có deselect `quickView.selectedQuickView` hay không. Tuy nhiên, cả Invoice và Claims `_applyView` đều **giống hệt** logic. Vấn đề thực sự nằm **sâu hơn**.

#### Invoice `_applyView` (line 3875, invoicePtE.js):
```javascript
function _applyView(closePanel, calculateTableSize) {
  // ... close panel logic ...
  if ($scope.searchTerms.isEdited) {
    $scope.quickView.selectedQuickView = null;
    _calculateWidthOfQuickView();
  }
  _getInvoices(null, null, false, false, calculateTableSize);
}
```

#### Claims `_applyView` (line 1704, insuranceClaimManagementATP.js):
```javascript
function _applyView(closePanel) {
  // ... close panel logic ...
  if ($scope.searchTerms.isEdited) {
    $scope.quickView.selectedQuickView = null;
    _calculateWidthOfQuickView();
  }
  getInsuranceClaimListSearchCriteria(null, null);
}
```

**Cả 2 đều giống nhau ở `_applyView`.**

## 🎯 ROOT CAUSE THỰC SỰ

### Luồng bug step-by-step:

```
Step 1: User remove column DOS → searchTerms.columns mất DOS
Step 2: Click Apply → _applyView(true)
  → searchTerms.isEdited CHƯA ĐƯỢC SET = true (vì isEdited chỉ check khi filter/column khác view gốc)
  → Nhưng thực tế, _applyView() gọi _getInvoices()
  → _getInvoices() set: headerDisplayColumns = cloneDeep(searchTerms.columns)
  → Table KHÔNG có DOS ✅ (đúng)
  
Step 3: Click Reset → _resetView()
  → _getView(viewId) → lấy view gốc từ server
  → _processSelectQuickView() → _renderViewToPanelFilter()
  → searchTerms.columns ĐƯỢC RESET về view gốc (CÓ DOS) ✅
  → searchTerms.isEdited = false ✅
  → _applyView(false) ĐƯỢC GỌI
  → _applyView kiểm tra: isEdited = false → KHÔNG NULL selectedQuickView ✅
  → _getInvoices() ĐƯỢC GỌI
  → headerDisplayColumns = cloneDeep(searchTerms.columns) → CÓ DOS ✅
```

**Theo logic trên, nó PHẢI hoạt động đúng.** Nhưng bug vẫn xảy ra. Vậy phải có thêm yếu tố gì đó...

### 🔎 Điều tra sâu hơn — `_resetView` trong Invoice (line 3624-3664):

```javascript
function _resetView() {
  if ($scope.searchTerms.viewId) {
    $scope.searchTerms.isEdited = false;
    _getView($scope.searchTerms.viewId).then((res) => {
      if (res) {
        $scope.searchTerms.isDefaultOfSystem = res.IsDefaultOfSystem;
        oldView.view._id = res.viewId;
        _processSelectQuickView(oldView.view, res);
        _applyView(false);  // ← GỌI APPLY SAU RESET
      }
    });
  } else {
    // TRƯỜNG HỢP KHÔNG CÓ viewId (untitled/edited view)
    $scope.quickView.selectedQuickView = defaultViewOfSystem._id;
    $scope.searchTerms.viewName = defaultViewOfSystem.Name;
    _renderViewToPanelFilter({...});
    // ⚠️ KHÔNG GỌI _applyView() !!!
  }
}
```

> [!IMPORTANT]
> ### Phát hiện Root Cause — Nhánh `else` của Invoice `_resetView`
> 
> Khi user thực hiện step 2 (click Apply mà không save), trạng thái `isEdited` sẽ thay đổi. Cần kiểm tra:
> 
> 1. **Sau step 2 (Apply)**, `searchTerms.viewId` có giá trị gì?
> 2. **`isEdited` flag** đã thay đổi chưa sau Apply?

### 🔑 Phân tích chính xác flow bug:

**Step 1-2**: User remove DOS column → click Apply
- `_applyView(true)` được gọi
- Kiểm tra `isEdited`:
  - `searchTerms.isEdited` **CHƯA được set = true** tại thời điểm click Apply vì user chỉ remove column mà không thông qua `_unselectColumn` trigger `isEdited`
  - **Nhưng wait...** function `_unselectColumn` KHÔNG set `isEdited = true`!

### ⭐ CRITICAL FINDING — `_unselectColumn()` KHÔNG set `isEdited`

#### Invoice `_unselectColumn` (line 3935):
```javascript
function _unselectColumn(column) {
  if ($scope.searchTerms.columns.length == 1) { return; }
  _.remove($scope.searchTerms.columns, (item) => item.Key === column.Key);
  $scope.searchTerms.allColumns.forEach((c) => {
    if (c.Key === column.Key) { c.selected = false; }
  });
  _sortColumnList();
  // ⚠️ KHÔNG SET isEdited = true
}
```

#### Claims `_unselectColumn` (line ~1567, trong insuranceClaimManagementATP.js):
```javascript
function _unselectColumn(column) {
  // ... tương tự ...
  // ⚠️ CŨNG KHÔNG SET isEdited = true
}
```

**Cả 2 đều không set `isEdited`!** Nhưng Claims hoạt động đúng sau Reset, còn Invoice thì không. **Vậy sự khác biệt phải nằm ở chỗ khác.**

## 🎯🎯 ROOT CAUSE CUỐI CÙNG

### So sánh flow gọi API khi Apply

#### Claims — `getInsuranceClaimListSearchCriteria()` (line 3507):
```javascript
$scope.claimTable.headerDisplayColumns = _.cloneDeep(
  $scope.searchTerms.columns  // ← LUÔN đồng bộ từ searchTerms.columns
);
```

#### Invoice — `_getInvoices()` (line 1429):
```javascript
$scope.invoiceTable.headerDisplayColumns = _.cloneDeep(
  $scope.searchTerms.columns  // ← CŨNG đồng bộ từ searchTerms.columns
);
```

**Cả 2 đều giống nhau!** Vậy nếu `searchTerms.columns` đúng → `headerDisplayColumns` phải đúng.

### 🔍 Kiểm tra lại flow step 2 (Apply) → step 4 (Reset):

> [!WARNING]
> **Giả thuyết mạnh nhất (95% khả năng)**:
> 
> Sau step 2 (Apply), `_applyView(true)` gọi `_getInvoices()`, hàm này set `headerDisplayColumns = searchTerms.columns` (KHÔNG CÓ DOS).
>
> Sau step 4 (Reset), `_resetView()` gọi `_getView()` (async) → `_processSelectQuickView()` → `_renderViewToPanelFilter()` set lại `searchTerms.columns` (CÓ DOS) → `_applyView(false)` → `_getInvoices()` → set `headerDisplayColumns = searchTerms.columns` (CÓ DOS).
>
> **Nhưng kiểm tra nhánh `else` trong `_resetView` Invoice**: Khi `searchTerms.viewId` rỗng (do `_applyView` set `selectedQuickView = null` khi `isEdited`), `_resetView` chạy vào nhánh `else` mà **KHÔNG gọi `_applyView()`** → Table KHÔNG được refresh!

### ✅ Xác nhận Root Cause:

Sau step 2 (Apply), nếu `searchTerms.isEdited` đã là `true` thì `_applyView()` set `selectedQuickView = null`. Nhưng **`searchTerms.viewId` vẫn giữ nguyên** (không bị null).

**Vậy nhánh `if ($scope.searchTerms.viewId)` vẫn đúng** → `_resetView` vẫn vào nhánh `if` → vẫn gọi `_applyView(false)`.

### 🔄 Quay lại kiểm tra — Claims `_resetView` vs Invoice `_resetView`:

So sánh 2 function, chúng **GIỐNG HỆT NHAU**:
- Cả 2 đều gọi `_getView()` → `_processSelectQuickView()` → `_applyView(false)`
- Cả 2 đều có nhánh `else` cho trường hợp không có `viewId`

> [!IMPORTANT]
> Code logic giữa Invoice và Claims cho Reset+Apply flow **gần như giống hệt nhau**. Bug có thể do:
> 1. **Timing issue** — `_getView()` async callback chưa complete trước khi `_getInvoices()` chạy
> 2. **$scope digest** — AngularJS digest cycle không trigger đúng trong Invoice context
