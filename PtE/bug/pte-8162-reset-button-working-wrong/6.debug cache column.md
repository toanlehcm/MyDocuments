### Khác biệt #1: Column Change Detection (CRITICAL!)

#### Invoice — Dùng `$watch` + closure variable

```javascript
// Line 393: Closure-level variable
let isChangeDisplayColumns = false;

// Line 486-503: $watch trên searchTerms.columns  
$scope.$watch("searchTerms.columns", function(newValue, oldValue) {
  if (oldView) {
    const stringifyOld = (oldView.columns || []).map(i => i.Key).join("");
    const stringifyNew = newValue.map(i => i.Key).join("");
    if (stringifyOld != stringifyNew) {
      isChangeDisplayColumns = true;   // ← SET TRUE khi columns khác oldView
    } else {
      isChangeDisplayColumns = false;  // ← SET FALSE khi columns giống oldView
    }
  }
});

// Line 1452: Trong _getInvoices, dùng closure variable
if (calculateTableSize || isChangeDisplayColumns) {
  setTimeout(() => {
    window.dispatchEvent(new Event("change-number-of-column"));
  }, 200);
} else {
  setTimeout(() => {
    window.dispatchEvent(new Event("resize"));  // ← CHỈ RESIZE, KHÔNG REFLOW COLUMN
  }, 100);
}
// ⚠️ KHÔNG CÓ cache update sau khi so sánh!
```

#### Claims — Dùng `cacheHeaderColumnVisible` + local comparison

```javascript
// Line 225: Closure-level cache
let cacheHeaderColumnVisible;

// Line 3522-3550: Trong getInsuranceClaimListSearchCriteria
let calculateTableSize = false, isChangeDisplayColumns = false; // ← LOCAL variable

const columnBefore = (cacheHeaderColumnVisible || []).map(i => i.Key).join('');
const columnAfter = ($scope.claimTable.headerDisplayColumns || []).map(i => i.Key).join('');

if (columnBefore !== columnAfter) {
  isChangeDisplayColumns = true;  // ← So sánh COLUMNS THỰC TẾ ĐANG HIỂN THỊ (cache) vs COLUMNS MỚI
}
if (String(cachePageSize) != String(pageSize)) {
  isChangeDisplayColumns = true;  // ← Thêm check pageSize change
}

if (calculateTableSize || isChangeDisplayColumns) {
  $timeout(() => {
    window.dispatchEvent(new Event("change-number-of-column")); // ← REFLOW COLUMN
  }, pageSize == 50 ? 1000 : pageSize == 20 ? 700 : 300);
} else {
  $timeout(() => {
    window.dispatchEvent(new Event("resize"));
  }, pageSize == 50 ? 1000 : pageSize == 20 ? 700 : 300);
}

// ⭐ QUAN TRỌNG: Update cache SAU KHI so sánh
cacheHeaderColumnVisible = _.cloneDeep($scope.claimTable.headerDisplayColumns);
cachePageSize = pageSize;
```

> [!CAUTION]
> ### Đây là ROOT CAUSE!
>
> **Invoice** so sánh `searchTerms.columns` vs `oldView.columns` qua `$watch`.
> Khi **Reset** gọi `_processSelectQuickView() → _renderViewToPanelFilter()`, columns được khôi phục = giá trị gốc từ server.
> `$watch` so sánh columns mới vs `oldView.columns` (cũng vừa được update bởi `_renderViewToPanelFilter`) → **GIỐNG NHAU** → `isChangeDisplayColumns = false`.
>
> Nhưng columns **thực tế đang hiển thị trên table** (headerDisplayColumns) vẫn là bộ cũ (thiếu DOS)!
> → `_getInvoices` kiểm tra `isChangeDisplayColumns = false` → chỉ dispatch `"resize"` → **TABLE KHÔNG REFLOW** → column DOS không xuất hiện!
>
> **Claims** so sánh `cacheHeaderColumnVisible` (columns đang hiển thị THỰC TẾ trên table) vs columns mới → **KHÁC NHAU** → `isChangeDisplayColumns = true` → dispatch `"change-number-of-column"` → **TABLE REFLOW** → column DOS xuất hiện!

### Khác biệt #2: Timer mechanism

| | Invoice | Claims |
|--|---------|--------|
| Timer | `setTimeout` (raw) | `$timeout` (AngularJS-aware) |
| Delay | Fixed `200ms` / `100ms` | Dynamic: `300ms` / `700ms` / `1000ms` tùy pageSize |

`$timeout` của Claims **tự động trigger digest cycle** sau khi callback chạy, đảm bảo AngularJS re-render DOM. `setTimeout` của Invoice **KHÔNG trigger digest** → có thể miss render update.

### Khác biệt #3: Scope apply

| | Invoice | Claims |
|--|---------|--------|
| Apply changes | `$timeout(() => { $scope.$apply() })` (line 1462) | `common.applyChanges($scope)` trong `.finally()` (line 3557) |
| Spinner off | Trong `.then()` callback | Trong `.finally()` block |

Claims dùng `.finally()` → spinner **LUÔN tắt** dù success hay error. Invoice tắt spinner trong `.then()` thành công VÀ error callback riêng.

## #2. Phân tích chi tiết `cacheHeaderColumnVisible` mechanism (Claims)

### Cơ chế hoạt động:

```
┌─────────────────────────────────────────────────────────────┐
│ Lần gọi API đầu tiên (init)                                │
│                                                             │
│ cacheHeaderColumnVisible = undefined                        │
│ columnBefore = [].join('') = ''                             │
│ columnAfter = 'DOSPatientNamePayerAmount...'                │
│ columnBefore !== columnAfter → isChangeDisplayColumns = true│
│ → dispatch "change-number-of-column" (full reflow)          │
│ → cacheHeaderColumnVisible = cloneDeep(headerDisplayColumns)│
└─────────────────────────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────────────────────────┐
│ Lần gọi API thứ 2 (Apply sau remove DOS)                   │
│                                                             │
│ cacheHeaderColumnVisible = [DOS, Patient, Payer, Amount...] │
│ columnBefore = 'DOSPatientNamePayerAmount...'               │
│ columnAfter = 'PatientNamePayerAmount...'  (thiếu DOS)      │
│ columnBefore !== columnAfter → isChangeDisplayColumns = true│
│ → dispatch "change-number-of-column" (full reflow)          │
│ → cacheHeaderColumnVisible = [Patient, Payer, Amount...]    │
└─────────────────────────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────────────────────────┐
│ Lần gọi API thứ 3 (Reset → _applyView(false))              │
│                                                             │
│ cacheHeaderColumnVisible = [Patient, Payer, Amount...]      │
│ columnBefore = 'PatientNamePayerAmount...'                  │
│ columnAfter = 'DOSPatientNamePayerAmount...' (CÓ DOS lại)  │
│ columnBefore !== columnAfter → isChangeDisplayColumns = true│
│ → dispatch "change-number-of-column" (full reflow) ✅       │
│ → cacheHeaderColumnVisible = [DOS, Patient, Payer, Amount..]│
│                                                             │
│ ⭐ Column DOS HIỂN THỊ LẠI ĐÚNG TRÊN TABLE!                │
└─────────────────────────────────────────────────────────────┘
```

### So sánh với Invoice (THIẾU cache):

```
┌─────────────────────────────────────────────────────────────┐
│ Lần gọi API thứ 3 trong Invoice (Reset → _applyView(false))│
│                                                             │
│ $watch đã fire:                                             │
│   oldView.columns = [DOS, Patient, Payer, Amount...]        │
│   searchTerms.columns = [DOS, Patient, Payer, Amount...]    │
│   stringifyOld === stringifyNew                             │
│   → isChangeDisplayColumns = FALSE ❌                       │
│                                                             │
│ _getInvoices kiểm tra: isChangeDisplayColumns = false       │
│ → dispatch "resize" (CHỈ resize, KHÔNG reflow column)      │
│ → Column DOS KHÔNG HIỂN THỊ! ❌                             │
└─────────────────────────────────────────────────────────────┘
```

> [!IMPORTANT]
> ### Kết luận Root Cause
>
> **Invoice** dùng `$watch` so sánh `searchTerms.columns` vs `oldView.columns` để quyết định `isChangeDisplayColumns`.
> Khi Reset khôi phục columns từ server, cả `searchTerms.columns` lẫn `oldView.columns` đều được update cùng giá trị (trong `_renderViewToPanelFilter`) → `$watch` thấy GIỐNG → `isChangeDisplayColumns = false` → table KHÔNG reflow → column bị mất vẫn không hiện lại.
>
> **Claims** dùng `cacheHeaderColumnVisible` — snapshot của columns **đang thực sự hiển thị trên table** — để so sánh. Cache chỉ update SAU KHI API response về. Nên khi Reset, cache vẫn là bộ cũ (thiếu DOS), columns mới là bộ đầy đủ → KHÁC NHAU → table REFLOW → column hiện lại.
>
> **Fix suggestion**: Invoice cần thêm mechanism `cacheHeaderColumnVisible` giống Claims, HOẶC ít nhất force `isChangeDisplayColumns = true` khi `_resetView` được gọi.
