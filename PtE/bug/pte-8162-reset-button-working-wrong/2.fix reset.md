function _getView(viewId) {

## Đánh giá Trade-off: Refactor `_resetView` theo ATP

### ⚠️ Scope thực tế — cần sửa 3 chỗ, không phải 1

| # | Thay đổi | Mức độ |
|---|----------|--------|
| 1 | Thêm function `_getView(viewId)` | Thêm mới (~10 dòng) |
| 2 | Sửa `_processSelectQuickView(view)` → thêm param `viewData` | Sửa function signature + logic |
| 3 | Sửa `_resetView()` → gọi `_getView` + truyền 2 params | Sửa nhánh `if (viewId)` |

**Lý do cần sửa #2:** ATP `_resetView` gọi `_processSelectQuickView(oldView.view, res)` truyền **2 params**. PtE hiện tại chỉ nhận **1 param** → nếu không sửa, `res` (server data) sẽ bị bỏ qua, hàm vẫn tìm trong `quickViewList` (stale).

### 📊 So sánh chi tiết

| Tiêu chí | PtE hiện tại (`oldView`) | ATP pattern (`_getView` API) |
|----------|--------------------------|------------------------------|
| **Data source** | `oldView.view` closure (snapshot cũ) | Server response (luôn mới nhất) |
| **Network call** | ❌ Không (instant) | ✅ 1 API call (`v4GetCustomViewDetail`) |
| **Stale data risk** | 🔴 Cao — `oldView` chỉ được set khi `_renderViewToPanelFilter` chạy | 🟢 Không — luôn fetch fresh |
| **UX speed** | ⚡ Instant reset | ⏳ Có spinner, đợi API (~200-500ms) |
| **Offline/lỗi mạng** | ✅ Vẫn hoạt động | ❌ Fail silently (`return undefined`) |
| **Multi-tab sync** | ❌ Không — nếu tab khác save, tab này reset về bản cũ | ✅ Có — fetch data mới từ server |
| **Số file thay đổi** | 0 | 1 file (`invoicePtE.js`) |
| **Impact radius** | - | `_processSelectQuickView` dùng ở 5+ chỗ — cần verify backward compatible |

### 🔍 Rủi ro khi sửa `_processSelectQuickView`

`_processSelectQuickView` được gọi ở **nhiều nơi** trong PtE:

| Nơi gọi | Cách gọi | Ảnh hưởng bởi thêm param? |
|---------|---------|---------------------------|
| `_resetView` | `_processSelectQuickView(oldView.view)` → sẽ đổi thành `(oldView.view, res)` | ✅ Cần sửa |
| `_deleteQuickView` | `_processSelectQuickView(defaultViewOfSystem)` | ❌ Không — param 2 undefined → fallback `quickViewList.find()` |
| `_selectQuickView` | `_processSelectQuickView(view)` | ❌ Không — tương tự |
| `_editQuickView` | `_processSelectQuickView(view)` | ❌ Không — tương tự |

**Kết luận:** Thêm param thứ 2 là **backward compatible** vì `if (viewData)` chỉ active khi truyền param 2. Các caller cũ truyền 1 param → `viewData = undefined` → vào nhánh `else` → giữ behavior cũ.
