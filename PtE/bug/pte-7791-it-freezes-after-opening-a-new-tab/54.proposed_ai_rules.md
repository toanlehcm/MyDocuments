# 📜 ĐỀ XUẤT CÁC QUY TẮC MỚI (AI RULES) TỪ BUG FREEZE

Từ bài học 2 tuần vật lộn với Bug Freeze, để AI (như em) trong các session sau trả lời "sắc bén", chuẩn kỹ thuật và không làm mất thời gian của anh, em đề xuất thêm các rules sau vào folder `.agent/quality-gates/` hoặc file `.rules` của dự án.

---

## RULE 1: ANTI YES-MAN & KỸ NĂNG "BẮT BỆNH" (Critical Thinking)

**Mô tả:** AI không được phép đoán mò hay ngoan ngoãn chèn code bừa bãi khi gặp lỗi "Đơ App" (Freeze/Hang) mà không có Error Log.

**Nội dung Rule cho AI:**
```markdown
# [UI Freeze Debugging Protocol]
Khi User báo lỗi "Đơ app", "Freeze", "Không click được" mà KHÔNG CÓ error log đỏ trong Console, AI TUYỆT ĐỐI KHÔNG ĐƯỢC đề xuất sửa code ngay. Bắt buộc phải yêu cầu User thực hiện 1 trong 2 bước dò tìm sau:
1. Khả năng Infinite Loop (Vòng lặp vô hạn): Yêu cầu User bật Chrome DevTools -> tab Performance -> Record 3 giây. Nếu CPU spike 100%, nguyên nhân là $digest loop.
2. Khả năng Z-index Lock (Đơ do UI đè ngầm): Yêu cầu User mở tab Elements, xóa thử thẻ overlay/mask đen (đặc biệt là mdDialog). Nếu xóa xong click được, nguyên nhân là Popup/Dialog config sai.
```

---

## RULE 2: ANGULARJS 1.X ROUTING & LIFECYCLE (Technical Accuracy)

**Mô tả:** Bắt lỗi ngay từ lúc AI (hoặc Dev) viết code điều hướng trang, tránh sinh ra lỗi $digest loop.

**Nội dung Rule cho AI:**
```markdown
# [AngularJS Routing Constraints]
Khi viết code hoặc review code có sử dụng `$location.path()` hoặc thay đổi/khởi tạo Route ($routeParams):
- KHÔNG BAO GIỜ đặt lệnh chuyển route ($location.path) đồng bộ (synchronous) ngay bên trong hàm khởi tạo controller (init/constructor) nếu nó phụ thuộc vào kết quả của chính route đó.
- BẮT BUỘC sử dụng `$timeout(function() { $location.path(...) }, 0);` để đẩy logic điều hướng ra khỏi vòng lặp $digest hiện tại, đưa vào Event Queue tiếp theo để tránh Infinite Digest Loop.
```

---

## RULE 3: NGĂN CHẶN "MEMORY LEAK" TỪ DIALOG

**Mô tả:** Angular Material rất hay bị lỗi kẹt Dialog nết config sai.

**Nội dung Rule cho AI:**
```markdown
# [Angular Material Dialog Safety]
Khi khởi tạo `$mdDialog.show(...)`:
- Tuyệt đối thận trọng với cờ `multiple: true` và `skipHide: true`. Mặc định KHÔNG sử dụng trừ khi có requirement rõ ràng bắt buộc mở 2 popup cùng lúc (Popup lồng Popup).
- Nếu phải dùng, bắt buộc phải viết kèm hàm cleanup để đóng dialog cũ trước khi mở cái mới, tránh tình trạng Z-index bị khóa và rác DOM.
```

---

## RULE 4: WEB SECURITY NATIVE (Noopener/Noreferrer)

**Mô tả:** Đảm bảo mọi tính năng mở tab mới do AI viết ra đều an toàn.

**Nội dung Rule cho AI:**
```markdown
# [External Link & Tab-opening Security]
Bất cứ khi nào viết DOM `<a target="_blank">` hoặc logic JS `$window.open(url, "_blank")`:
- BẮT BUỘC phải đính kèm `rel="noopener noreferrer"` (với HTML) hoặc `windowFeatures: "noopener,noreferrer"` (với JS).
- Không được bỏ qua lớp bảo mật này kể cả khi mở link tải file (Download), nhằm ngăn chặn tấn công Reverse Tabnabbing.
```
