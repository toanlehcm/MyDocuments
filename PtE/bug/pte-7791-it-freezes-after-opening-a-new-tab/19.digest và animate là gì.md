## Trả lời câu 1: `$digest` và `$animate` là gì?

Đây là **cơ chế bên trong của AngularJS** (framework), nằm trong file thư viện `angular.js` — KHÔNG nằm trong code dự án. Anh không cần tìm.

| Cơ chế | Giải thích đơn giản |
|--------|---------------------|
| **$digest** | "Vòng kiểm tra" — mỗi khi data thay đổi, AngularJS chạy 1 vòng kiểm tra TẤT CẢ bindings (ng-model, ng-show, ng-if...) để cập nhật UI. Nếu data thay đổi liên tục → vòng chạy liên tục → UI bị chậm/freeze. |
| **$animate** | "Bộ animation" — khi ng-show thêm/bỏ class `ng-hide`, AngularJS gọi `$animate` để kiểm tra xem element có CSS animation không. Nó gọi `getComputedStyle()` trên MỖI element → nếu nhiều element → trình duyệt phải tính layout nhiều lần → chậm. |

**Tóm gọn:** Code của mình không gọi trực tiếp, nhưng mỗi lần `ng-show`, `ng-model`, `ng-repeat` chạy → AngularJS tự động dùng `$digest` + `$animate` đằng sau. Khi quá nhiều → freeze.

## Áp dụng 2 Fix:

**Fix 1:** Thêm `$animateProvider.classNameFilter` — bảo AngularJS **CHỈ** check animation cho elements có class `md-*` (Angular Material), SKIP tất cả elements khác → triệt tiêu layout thrashing.

**Fix 2:** Sửa dropdown KHÔNG ghi đè `ngModel` từ `null` → `""` — giảm $digest storm từ 32 iterations xuống 2-3.
