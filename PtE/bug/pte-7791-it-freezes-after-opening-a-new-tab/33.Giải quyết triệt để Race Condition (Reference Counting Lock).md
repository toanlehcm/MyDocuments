# Giải quyết triệt để Race Condition (Reference Counting Lock)

Bản thiết kế này mô tả phương án chốt "khóa an toàn" lên global spinner (`$rootScope.spinnerCounter`) nhằm điều phối chuẩn xác thời điểm compile `<md-contact-chips>`. Phương án này triệt tiêu hoàn toàn rủi ro **Spinner Flicker** (chớp tắt màn Loading) và **Race Condition** (Mạng chậm).

## Ưu nhược điểm (Trade-offs)

| Tiêu chí | Cấu trúc Reference Counting Lock |
| :--- | :--- |
| **Bản chất Time Complexity** | **O(1)** đối với kiến trúc Locking. Khối lượng tính toán (Compile + DOM Reflow) của nội bộ Component `md-contact-chips` không hề thay đổi, nhưng nó được phân bổ thực thi vào "Freeze Window" an toàn (đằng sau màn che). |
| **UX (Trải nghiệm UI)** | **Hoàn hảo.** Không chớp nháy (No Flicker). Không giựt cục (No Layout Shift). Từ góc nhìn của User, màn hình Loading quay lâu hơn 1.5 giây, nhưng khi Loading vụt tắt thì UI rỗng 100% CPU, thao tác F5 hay Click `+ Task` phản hồi lập tức. Độ êm ái đạt MAX. |
| **Rủi ro rò rỉ (Edge Case)** | Nếu `Promise.all` sinh lỗi văng Exception (Reject) mà ta không có khối `.catch()`, biến `$rootScope.spinnerCounter--` sẽ không được gọi $\rightarrow$ Loading Spinner xoay vĩnh viễn. Đòi hỏi phải bọc `.catch()` hoặc `finally()`. |

---

## Proposed Changes

### Thay đổi Cấu trúc View & Controller

#### [MODIFY] `tasksManagementPtE.js`
Thiết lập lại hàm `initializer()` theo mô hình State-Machine chặn đứng Network và Spinner.

```javascript
function initializer() {
    $scope.initializer = true;
    
    // 1. Chốt khóa an toàn (Tiêm ống thở)
    $rootScope.spinnerCounter = ($rootScope.spinnerCounter || 0) + 1;
    $scope.isPatientChipsReady = false;

    // ... (logic giấu Nav Bar trên mobile giữ nguyên) ...

    Promise.all([loadTaskUserPreferences(), loadTaskGeneralData()])
        .then(() => {
            initFilterDefault();
            
            // 2. Network xong -> Gạt cờ thả DOM ra compile
            $scope.isPatientChipsReady = true;
            
            // 3. Đợi DOM tiêu hóa đợt Reflow khổng lồ rồi mới tháo khóa
            $timeout(function() {
                if ($rootScope.spinnerCounter > 0) {
                    $rootScope.spinnerCounter--; 
                }
            }, 1000); // 1 giây là quá đủ cho cục Reflow
        })
        .catch((err) => {
            // Safety: Tháo khóa nếu API văng lỗi
            console.error(err);
            if ($rootScope.spinnerCounter > 0) $rootScope.spinnerCounter--;
        });
}
```

#### [KEEP] `tasksManagementPtE.html`
- Thẻ `<div class="col-12" ng-if="isPatientChipsReady">` đã được áp dụng ở bản trước. Không cần phải sửa đổi thêm ở cấp độ HTML. Lệnh `ng-if` sẽ tự động trigger mount ngay khi Promise `.then()` trả về cờ true.

## Open Questions

> [!CAUTION]
> Chú ý cơ chế Catch Error: Giải pháp "Khóa bộ đếm" (Counter Locking) có nhược điểm chí mạng là **phải gỡ khóa bằng mọi giá**. Em đã tích hợp block `.catch()` để đề phòng API chết thì Spinner vẫn rớt về 0. Anh đồng ý với chốt chặn an toàn này không?
