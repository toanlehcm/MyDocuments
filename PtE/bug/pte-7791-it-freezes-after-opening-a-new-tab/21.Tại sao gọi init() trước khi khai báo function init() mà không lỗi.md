Câu hỏi hay. Giải thích ngắn gọn:

## Tại sao gọi `init()` trước khi khai báo `function init()` mà không lỗi?

Vì JavaScript có **hoisting** — `function init() { ... }` được "nâng" lên đầu scope tự động. Nên gọi trước hay sau đều chạy đúng.

```javascript
// JavaScript engine "hiểu" code như thế này:
// ┌── function init() được hoisted lên đây ──┐
// │                                            │
init();  // ← gọi được vì init đã hoisted     │
//                                              │
function init() { ... }  // ← khai báo gốc ────┘
```

## Nhưng nếu anh muốn đặt cuối file cho sạch, hoàn toàn được

Cả 2 cách đều an toàn. Nếu anh thích đặt cuối controller như code gốc ban đầu, anh cứ move dòng `init();` xuống cuối — **không ảnh hưởng gì**. Đây chỉ là code style preference, không phải technical issue.
