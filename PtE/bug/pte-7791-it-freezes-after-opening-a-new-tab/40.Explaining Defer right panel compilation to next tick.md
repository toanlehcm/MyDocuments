```js
function initializer() {
            $scope.initializer = true;
            $scope.filterPanelReady = false;

            if ($rootScope.screenType === ScreenTypeConst.Phone && $scope.flagshowPanel) {
                $rootScope.hideHeaderLogin = true;
                $rootScope.hideNav = true;
                $('.top-ul').hide();
            }

            Promise.all([loadTaskUserPreferences(), loadTaskGeneralData()]).then(() => {
                initFilterDefault();

                // Defer right panel compilation to next tick. Reduce animation callbacks in the first render (route transition).
                $timeout(function() {
                    $scope.filterPanelReady = true;
                }, 0);
            });
        }
```

Trong đoạn code trên, quá trình chia làm 2 giai đoạn rất rõ ràng nhờ kỹ thuật **"Cố tình trì hoãn" (Defer rendering)**. Dưới đây là mốc thời gian chính xác:

### 1. Dữ liệu (Data) có khi nào?
Dữ liệu có **trước**. 
Cụ thể là ngay khi 2 API (`loadTaskUserPreferences` và `loadTaskGeneralData`) cùng chạy xong $\rightarrow$ Nhảy vào `.then()` $\rightarrow$ Hàm `initFilterDefault()` chạy.
Tại lúc này, toàn bộ dữ liệu bộ lọc (Filter Data) đã nằm ngay ngắn trong RAM (trong các biến `$scope`), TẤT CẢ chức năng đã sẵn sàng phục vụ.

### 2. Giao diện (DOM) được vẽ (Render) ra khi nào?
Giao diện được render **sau đó 1 nhịp (1 tick)**.
Mặc dù dữ liệu đã có đủ rồi, nhưng thẻ `div` của Right Panel vẫn chưa hề tồn tại trong DOM (vì lúc này `$scope.filterPanelReady` vẫn đang là `undefined` hoặc `false` ở lần khởi tạo rỗng, nên `ng-if` chặn không cho vẽ).

Chỉ khi câu lệnh này được gọi:
```javascript
$timeout(function() {
    $scope.filterPanelReady = true;
}, 0);
```
**Bản chất của `$timeout` với delay bằng `0` là:** *"Ê Trình duyệt, anh đang bận vẽ các phần chính (danh sách công việc, header, animation chuyển trang). Cứ thư thả vẽ cho xong đi. Bao giờ CPU rảnh rang (vào chu kỳ event loop tiếp theo), hãy quay lại đây gạt công tắc `filterPanelReady = true` nhé."*

Lúc công tắc này được bật, một vòng Digest Cycle mới của Angular mới được kích hoạt, thẻ `<div ng-if="...` lọt qua cửa kiểm duyệt, và lúc này toàn bộ khối HTML của Right Panel mới thực sự được tiêm (inject) vào DOM và bốc dữ liệu ra hiển thị.

👉 **Tóm lại:** Data có ngay lập tức khi API xong. Nhưng DOM thì phải "bước lùi lại một nhịp" nhường cho các Animation hoặc phần View gốc Render xong xuôi, thì nó mới bắt đầu được nạp vào để tránh giật lag (Bottleneck) ở giây đầu tiên vào trang!
