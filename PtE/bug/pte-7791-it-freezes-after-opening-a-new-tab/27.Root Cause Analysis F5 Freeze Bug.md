## 🎯 Root Cause Gốc Rễ

### Freeze KHÔNG do thiếu `$rootScope` state
(Đã chứng minh: CRITICAL state giống nhau ở cả 2 case)

### Freeze do 3 nguyên nhân ĐỒNG THỜI:

#### 1. 🌐 Network Contention (Tranh giành kết nối mạng)

Sau F5, **Tab 1 (Calendar) VÀ Tab 2 (Tasks)** cùng gọi API đến cùng 1 server:

| Tab | API calls đồng thời |
|-----|---------------------|
| Tab 1 | v2Authenticate, v3GetDataSession, schedulerData, appointments (7 ngày), WebSocket |
| Tab 2 | v3GetDataSession, v4GetPublicSystemConfig, v4GetAllStateByCountryATP, v4GetTaskGeneralData, WebSocket |
| **Tổng** | **~15-20 API calls cùng lúc** |

> HTTP/1.1 chỉ cho phép **6 connection đồng thời per domain**. Tab 2 phải ĐỢI Tab 1 giải phóng connection → API response chậm → render chậm.

**Bằng chứng:** `WebSocket connection FAILED: WebSocket is closed before the connection is established` ← Tab 2 không đủ connection để mở WebSocket.

#### 2. ⚡ Main Thread Contention (Tranh giành CPU)

Chrome đặt các tab cùng domain vào **cùng 1 renderer process** (khi `noopener` nhưng same-site). Cả 2 tabs chia sẻ 1 main thread:

- Tab 1: dhtmlx Scheduler đang render (parse events, draw SVG, layout calculation)
- Tab 2: AngularJS đang compile templates, run digest cycles
- `[ScopeCleaner]` chạy **15+ lần** ← mỗi lần là 1 digest cycle trên main thread
- Khi user click dropdown → Angular cần: compile template + $digest + bind-html-compile cho từng option → **MAIN THREAD ĐÃ BẬN → FREEZE**

#### 3. 🔄 Heavy Initialization Cascade

Tab 2 mới (noopener,noreferrer) phải **làm lại TẤT CẢ từ đầu**:

```
1. Load & parse ALL JS files (AngularJS, Material, lodash, moment, dhtmlx...)
2. Bootstrap Angular module
3. Gọi v3GetDataSession → initDataSession (55+ $rootScope vars)
4. Kết nối WebSocket (FAILED → retry)
5. [ScopeCleaner] x15 (digest cleanup)
6. Load task data
7. Compile popup template
8. User click dropdown → Compile dropdown template → FREEZE
```

## 💡 Solution

### Tại sao đây là Edge Case hợp lệ để bỏ qua?

1. **User hiếm khi nhấn F5** — trong workflow thực tế, user mở app 1 lần và dùng cả ngày
2. **F5 + mở new tab NGAY LẬP TỨC** — workflow cực kỳ hiếm gặp
3. **Fix toàn diện = rewrite architecture** — SPA legacy + shared renderer process = không thể fix hoàn toàn

### Giải pháp khắc phục (nếu leader yêu cầu):

#### Option A: Thêm loading state cho dropdown (Low effort)
Thêm spinner/skeleton cho dropdown khi Tab 2 chưa hoàn toàn sẵn sàng — popup mở nhưng dropdown hiện "Loading..." thay vì freeze.

#### Option B: Debounce popup initialization (Medium effort)
Trong `popupAddEditTaskPtE.js`, đợi `requestIdleCallback` trước khi set `$scope.initialized = true`. Đảm bảo main thread idle trước khi render dropdown.

#### Option C: Open Tab 2 trong setTimeout (Low effort)
Trong `viewAssociatedTasks()` (app.js:4964), thay `window.open()` bằng `setTimeout(() => window.open(...), 100)`. Cho Tab 1 thêm thời gian settle trước khi Tab 2 mở.

### ✅ Đã implement (từ session trước):
- `ng-if="initialized"` cho dropdown fields — popup mở ngay, dropdown lazy render
- Optimized `dropdownListPtE.js` — giảm cascading digest cycles
