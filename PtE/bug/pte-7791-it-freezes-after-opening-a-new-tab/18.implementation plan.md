# 🐛 Phân Tích Root Cause: UI Freeze Trên Tasks Module

## Bối cảnh

Có **2 case freeze riêng biệt**, cần phân tích từng case độc lập.

---

## CASE 1: Freeze ở tab thứ 7+

### Triệu chứng
- Click "View associated task" mở new tab nhiều lần
- Từ tab thứ 7 trở đi, tab mới bị freeze

### Phân tích workflow (code trace)

```
Click "View associated task"
  → viewAssociatedTasks() [app.js:4972]
    → commonService.openInternalLink(`/tasks/${patientId}/${relatedTo}/${id}`, true) [app.js:4974]
      → $window.open(link, "_blank", "") [commonService.js:1933]
        → New tab mở CÙNG PROCESS với tab gốc (KHÔNG có noopener)
```

**Tại sao freeze ở tab 7+?**

Khi `window.open()` KHÔNG có `noopener`:
- Tất cả tabs chia sẻ **cùng 1 browser process** (renderer process)
- Mỗi tab tạo:
  - 1 AngularJS app instance (100+ `$rootScope` watchers)
  - 1 WebSocket connection ([actionheroFactory.js:597](file:///d:/Sources/pteverywhere/Client/app/scripts/factory/actionheroFactory.js#L597))
  - Hàng trăm `$watch` (dropdown directives, ng-repeat, etc.)
  - `$interval` timers từ mỗi dropdown ([dropdownListPtE.js](file:///d:/Sources/pteverywhere/Client/app/scripts/directives/dropdownListPtE.js))
- Mỗi tab cũng increment `applicationCount` trong localStorage ([sessionFactory.js:1479](file:///d:/Sources/pteverywhere/Client/app/scripts/factory/sessionFactory.js#L1479))
- Từ tab 7+: tổng **7 × (watchers + WS + timers)** share 1 CPU core → main thread exhausted

### Fix đã áp dụng
```javascript
// commonService.js:1932
var windowFeatures = isBlankTab ? "noopener,noreferrer" : "";
$window.open(link, isBlankTab ? "_blank" : "_self", windowFeatures);
```

**Đánh giá:** Fix này ĐÚNG, mỗi tab có process riêng → không còn share CPU → fix triệt để case 1.

#### Bước 6: sessionFactory() hai lần (CRITICAL)

**Lần 1** (route `/tasks/:params`):
```javascript
// sessionFactory.js:1446 - isLogin = undefined (false)  
// sessionFactory.js:1470 - $rootScope.isFirstLoad = true ← YES (new tab)
//   → increment applicationCount [1479] → localStorage("applicationCount") = 1
//   → go to else branch (isLogin = false) [1534]
//   → applicationCount (1) <= 1 && !sessionData (null) ← TRUE!
//   → $rootScope.clearDataSession() → REDIRECT TO HOME PAGE!!!
```

> [!CAUTION]
> **ROOT CAUSE TÌM THẤY Ở ĐÂY!**
>
> Khi new tab mở với `noopener`:
> - `sessionStorage` = **TRỐNG** (noopener tạo new browsing context, KHÔNG copy sessionStorage)
> - `applicationCount` trong localStorage = **0** hoặc **1** (vì logout đã reset về 0, tab mới increment lên 1)
> - Condition `applicationCount <= 1 && !sessionData` = **TRUE**
> - → Gọi lại `clearDataSession()` → redirect home → **controller đang init bị gián đoạn**
>
> **Nhưng** vì `localStorage.LOGIN_IN_DATA` VẪN CÒN (set lúc login ở tab gốc), nên route chuyển về login → phát hiện đã đăng nhập → redirect lại `/tasks` → **VÒng lặp hoặc race condition!**

### Kiểm chứng giả thuyết

**Tại sao KHÔNG freeze khi không logout trước?**

Khi không logout:
- Tab gốc đã set `applicationCount` > 1 (từ nhiều lần init)
- sessionStorage ở tab gốc CÓ data
- New tab (noopener): sessionStorage trống, NHƯNG applicationCount > 1
- → Condition `applicationCount <= 1 && !sessionData` = **FALSE** (vì count > 1)
- → Đi vào nhánh timeout check → session chưa timeout → gọi API v3GetDataSession → **HOẠT ĐỘNG BÌNH THƯỜNG**

**Tại sao freeze KHÔNG ổn định (lúc được lúc không)?**

Race condition giữa:
- `applicationCount` depends on timing: nếu tab gốc đã increment count > 1 trước khi new tab đọc → có thể pass condition
- Nếu new tab đọc count nhanh hơn → count = 0 hoặc 1 → fail condition → freeze

## Root Cause Chính Xác

### Case 1 ✅ (Đã fix đúng)
**Browser process sharing** → quá nhiều watchers/timers share 1 CPU core
**Fix:** `noopener,noreferrer` → process isolation

### Case 2 ❌ (Chưa fix)
**`sessionStorage` isolation do `noopener` + logic kiểm tra session dựa trên `applicationCount` và `sessionStorage`**

Cụ thể, [sessionFactory.js line 1535](file:///d:/Sources/pteverywhere/Client/app/scripts/factory/sessionFactory.js#L1535):
```javascript
if(applicationCount <= 1 && !sessionData){
    $rootScope.clearDataSession();
    $rootScope.redirectToTheHomePage();
}
```

Logic này giả định:
- Nếu `applicationCount <= 1` (ít tab) VÀ `sessionStorage` trống → user chưa login hoặc session hết hạn → kick về login
- **NHƯNG** khi dùng `noopener`, `sessionStorage` LUÔN trống ở tab mới, dù user đã login
- Kết hợp với `applicationCount` bị reset về 0 sau logout → condition LUÔN TRUE sau logout+login+new tab

Kết quả: `clearDataSession()` bị gọi trong khi controller đang init → **state corruption** → freeze hoặc redirect loop.

## Proposed Fix

### Fix cho Case 2: Sửa logic kiểm tra session trong new tab

**Vấn đề cốt lõi:** Logic session check dựa trên `sessionStorage` (per-tab) + `applicationCount` (shared localStorage), nhưng `noopener` khiến `sessionStorage` luôn trống ở new tab.

**Fix đề xuất:** Khi `localStorage.LOGIN_IN_DATA` tồn tại (user đã login), KHÔNG nên kick user về login page chỉ vì `sessionStorage` trống. Thay vào đó, dùng `localStorage` data + API `v3GetDataSession` để khôi phục session.

#### [MODIFY] [sessionFactory.js](file:///d:/Sources/pteverywhere/Client/app/scripts/factory/sessionFactory.js)

Sửa logic ở line 1534-1544:

```diff
- if(applicationCount <= 1 && !sessionData){
-     $rootScope.clearDataSession();
-     $rootScope.redirectToTheHomePage();
-     if(!$rootScope.$root.$$phase) {
-         $rootScope.$apply();
-     }
- }
+ if(applicationCount <= 1 && !sessionData && !userData){
+     // Only redirect to home if BOTH sessionStorage AND localStorage are empty
+     // (user truly not logged in). With noopener, sessionStorage is always
+     // empty in new tabs, so we must also check localStorage.
+     $rootScope.clearDataSession();
+     $rootScope.redirectToTheHomePage();
+     if(!$rootScope.$root.$$phase) {
+         $rootScope.$apply();
+     }
+ }
```

**Giải thích:** `userData` (line 1482-1486) đã được đọc từ `localStorage.LOGIN_IN_DATA`. Nếu `userData` tồn tại, user đã login → KHÔNG kick về home. Chỉ kick khi CẢ 3 điều kiện: count thấp, sessionStorage trống, VÀ localStorage trống.
