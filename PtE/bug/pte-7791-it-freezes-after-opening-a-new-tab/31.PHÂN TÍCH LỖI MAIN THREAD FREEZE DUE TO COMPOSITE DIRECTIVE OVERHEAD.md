### 1. Giả thuyết của anh có đúng không?
**Hoàn toàn chính xác (100%).** `md-contact-chips` chính là Blocker trực tiếp đánh sập Main Thread.

**Cơ sở kỹ thuật (Tại sao?):**
Bản thân `<md-contact-chips>` không phải là một node lá đơn thuần. Nó là một **Composite Directive** cực kỳ cồng kềnh, hoạt động dựa trên việc wrap (kết nối ngầm) 3 core directives khác của Angular Material: 
1. `md-chips` (quản lý state các thẻ).
2. `md-autocomplete` (quản lý input lookup).
3. `md-virtual-repeat` (quản lý rendering danh sách patient dropdown).
Khi anh load component này, Engine của Angular 1.x phải chạy quá trình `Compile -> Link -> Bootstrap` cho toàn bộ cây DOM con khổng lồ này, khiến Call Stack bị bão hòa (Saturation) đúng vào thời điểm `$mdDialog` (Popup) đang cố gắng trigger CSS Transition animation. Sự tranh chấp CPU cycle này gây ra hiện tượng UI ngưng trệ.

### 2. Ba (03) Giả Thuyết Về Cơ Chế Gây Nghẽn (Root Cause)

🎯 **Giả thuyết A (70%): Layout Thrashing (Forced Synchronous Layout)**
- **Nguyên nhân:** Trong hàm `link()` của module `md-autocomplete` ngầm dưới component này, Material Core thực thi liên tục các lệnh đọc DOM metric như `getBoundingClientRect()`, `offsetWidth` để định cỡ Input box và Dropdown Wrapper.
- **Cơ chế nghẽn:** Đọc DOM property khi DOM Tree đang bị mutates (Right Panel chuẩn bị paint) ép browser phải dừng xử lý JS để thực hiện **Reflow** toàn cục ngay trong frame hiện tại. Chuỗi Reflow liên hoàn này bóp nghẹt 60fps frame rate của browser, chặn đứng `$mdDialog.show()`.

🎯 **Giả thuyết B (20%): Digest Cycle Explosion (Quá tải Dirty Checking)**
- **Nguyên nhân:** Do thuộc tính `ng-model="filterModel.Patients"` kết hợp với chuỗi flag `filter-selected="true"` và `md-require-match="true"`.
- **Cơ chế nghẽn:** Để duy trì ràng buộc phức tạp (deep object array), directive này tự đăng ký hàng chục `$watchCollection` hoặc `$watch(..., true)`. Khi controller nạp data list (`onChangePatients` / scope thay đổi), Digest Loop của Angular phải lặp qua lặp lại việc so sánh vùng nhớ các object Patient, đẩy thời lượng Execute JS vượt mốc 500ms (Max threshhold của 1 Task trên thread).

🎯 **Giả thuyết C (10%): `md-virtual-repeat` Eager Compilation**
- **Nguyên nhân:** Node list xổ xuống của Autocomplete cố compile sẵn các template ngầm dù User chưa focus vào ô Input.
- **Cơ chế nghẽn:** Thay vì lazy-compile, Angular instantiate scope và nội suy DOM element cho dropdown, tiêu tốn extra Memory và CPU cycles dồn dập vào cùng thời điểm khởi tạo Right Panel.
