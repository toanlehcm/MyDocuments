# 🔥 REAL ROOT CAUSE ANALYSIS — UI Freeze in New Tab

> **Kết luận:** Freeze KHÔNG phải do Tab 1 vs Tab 2, KHÔNG phải do shared main thread, KHÔNG phải do popup code.  
> **Root cause thật sự:** TAB 2 TỰ FREEZE CHÍNH NÓ bởi **"Digest Storm"** — một chuỗi phản ứng dây chuyền từ 4 thủ phạm trong code.

## ✅ ROOT CAUSE THẬT SỰ: "Digest Storm" — Tab 2 tự freeze chính nó

### 4 Thủ phạm tạo nên Digest Storm

---

### 🔴 THỦ PHẠM #1: `safeBindHtml.js` — NGHIÊM TRỌNG NHẤT

**File:** `scripts/directives/safeBindHtml.js`

```javascript
// Line 41-70: Decorator thay thế TOÀN BỘ ng-bind-html trong app
scope.$watch(
  function() { /* evaluate expression MỖI digest */ },
  function(htmlString) {
    const safeHtml = domPurifyService.sanitize(String(htmlString)); // CPU-heavy!
    console.log('safeHtml', safeHtml); // ⚠️ LINE 62: LOG MỖI DIGEST!
    if (safeHtml !== lastSafeHtml) {
      element.html(safeHtml);
    }
  }
);
```

**Tác hại:**
- Decorator này **thay thế** toàn bộ `ng-bind-html` directive trong app
- App có **98+ file HTML** sử dụng `ng-bind-html`
- Mỗi page có thể có **10-50+** element dùng `ng-bind-html`
- Mỗi element tạo **1 watcher** → chạy MỖI digest cycle
- Mỗi watcher gọi `domPurifyService.sanitize()` (CPU-heavy) + `console.log()` (I/O block)
- **10 digest cycles × 30 elements = 300 lần console.log + 300 lần DOMPurify**

> [!CAUTION]
> `console.log('safeHtml', safeHtml)` ở line 62 là **DEBUG LOG CHƯA GỠ**, chạy trong PRODUCTION!
> Đây là log anh thấy trong console: `safeHtml Auto-payment failed: Invoice...` (20-30 dòng)

### 🔴 THỦ PHẠM #2: `actionheroFactory.js` — WebSocket gây $apply storm

**File:** `scripts/factory/actionheroFactory.js`

```javascript
// Line 81: newmessage handler — KHÔNG check $$phase!
case 'newmessage':
    onNewMessageListener({...});
    $rootScope.$apply();  // ⚠️ KHÔNG check $$phase → crash nếu digest đang chạy

// Line 93: systemmessage handler — KHÔNG check $$phase!
case 'systemmessage':
    onSystemMessageListener(messageInfo);
    $rootScope.$apply();  // ⚠️ KHÔNG check $$phase

// Line 251-253: notify-system handler
setTimeout(function() {
    $rootScope.$apply();  // ⚠️ KHÔNG check $$phase, delay chỉ 10ms
}, 10);
```

**So sánh:** Các handler khác (insurance-claim-form, insurance-bulk-claim, etc.) ĐỀU có check:
```javascript
if (!$rootScope.$root.$$phase) {
    $rootScope.$apply();  // ✅ An toàn
}
```

**Tác hại:**
- Khi login xong → WebSocket connect → server gửi **queued messages** (backlog)
- Mỗi message → `$rootScope.$apply()` → trigger FULL digest cycle
- Nếu gọi `$apply()` trong khi digest đang chạy → error hoặc silent fail
- 10 messages liên tục = **10 digest cycles liên tục** → main thread bận

## ❓ Tại sao chỉ freeze khi logout→login nhưng KHÔNG freeze khi chỉ F5?

| Factor | Logout → F5 → Login | Chỉ F5 (không logout) |
|--------|---------------------|----------------------|
| WebSocket | Disconnect → Reconnect mới → Server gửi **QUEUED messages** | Reconnect cùng session → **ÍT messages** |
| initDataSession | Chạy FULL (set 55+ $rootScope vars) | Chạy FULL nhưng session gần giống |
| Server notification backlog | **CAO** (messages tích lũy khi offline) | **THẤP** (offline rất ngắn) |
| Digest cycles từ WebSocket | **NHIỀU** (10-20 messages liên tục) | **ÍT** (0-3 messages) |
| safeBindHtml executions | **NHIỀU** × số elements × số digests | **ÍT** |

---

## ❓ Tại sao production "lúc bị lúc không"?

1. **Phụ thuộc số lượng queued messages** — Nếu server ít notification → ít $apply → không freeze
2. **Network speed** — Mạng chậm → messages đến rải rác → digest có thời gian settle
3. **Tốc độ user click** — Click chậm → background work xong → không freeze
4. **Số ng-bind-html elements** — Page có ít elements → ít watcher → nhẹ hơn
