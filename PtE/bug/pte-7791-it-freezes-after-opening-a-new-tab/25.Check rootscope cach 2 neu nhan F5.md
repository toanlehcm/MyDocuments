## Trả lời: Project KHÔNG có thư viện state management

### 1. Kết quả check package.json

**KHÔNG có bất kỳ thư viện state management nào** tương tự Redux/Zustand. Dependencies chỉ có:

### 2. AngularJS 1.x "state management" là gì?

AngularJS 1.x **KHÔNG CÓ** state management library riêng như Redux. Thay vào đó, project này dùng **3 cơ chế tự dựng**:

| Cơ chế | Tương đương React | Vị trí trong project | F5 mất? |
|--------|-------------------|---------------------|---------|
| **`$rootScope`** | Redux global store | `sessionFactory.js` line 138-800+, `app.js` | ✅ **MẤT 100%** |
| **Services/Factories** (singleton) | React Context providers | `sessionFactory.js`, `taskFactory.js`, `CommonFactory.js` | ✅ **MẤT 100%** |
| **`sessionStorage`** | localStorage persist | Lưu PUBLIC_SYSTEM_CONFIG, login data | ⚠️ Còn nếu cùng tab, **MẤT** nếu logout xóa |

```
So sánh React vs AngularJS 1.x:

React + Redux:
┌─────────────────────────┐
│  Redux Store (global)   │ ← F5 MẤT (trừ khi dùng redux-persist)
│  ├── user state         │
│  ├── clinic state       │
│  └── UI state           │
└─────────────────────────┘

AngularJS 1.x (project này):
┌─────────────────────────┐
│  $rootScope (global)    │ ← F5 MẤT 100% — giống Redux store
│  ├── userId             │
│  ├── timeZoneValue      │
│  ├── dateFormat         │
│  ├── icons[]            │
│  ├── clinicId           │
│  └── 100+ vars nữa...  │
├─────────────────────────┤
│  Factory singletons     │ ← F5 MẤT 100% — giống React Context
│  ├── sessionFactory     │
│  ├── taskFactory        │
│  └── CommonFactory      │
└─────────────────────────┘
```

### 3. Cách check $rootScope bị mất khi F5

Anh mở Console trên trang Tasks và chạy:

```javascript
// ===== SCRIPT ĐO STATE =====
var $root = angular.element(document.body).scope().$root;
var stateKeys = Object.keys($root).filter(k => !k.startsWith('$') && !k.startsWith('_'));

console.log('=== STATE MANAGEMENT CHECK ===');
console.log('Tổng số state vars:', stateKeys.length);
console.log('');

// Check các state QUAN TRỌNG (giống Redux selectors)
var criticalState = {
  'userId': $root.userId,
  'clientId': $root.clientId,
  'userRoleSelected': $root.userRoleSelected,
  'timeZoneValue': $root.timeZoneValue,
  'dateFormat': $root.dateFormat,
  'token': $root.token ? '✅ Có' : '❌ Không',
  'icons': $root.icons ? $root.icons.length + ' items' : '❌ Không',
  'ClinicName': $root.ClinicName,
  'fullName': $root.fullName,
  'insurancePermission': $root.insurancePermission ? '✅ Có' : '❌ Không'
};

console.table(criticalState);
console.log('Timestamp:', new Date().toISOString());
```

   - Nếu đợi 5 giây: State đã rebuild xong → giống B1 ✅

### 5. Kết luận cho leader

> **"Project AngularJS 1.x này dùng `$rootScope` làm state management (tương tự Redux store). Khi nhấn F5, toàn bộ `$rootScope` (~100+ state variables) bị destroy hoàn toàn. Sau khi login, app phải rebuild TẤT CẢ state qua API calls — quá trình này mất 2-5 giây (hàm `initDataSession()` ~400 lines code, gán 100+ biến lên $rootScope). Nếu user mở tab mới trước khi state rebuild xong, tab mới cũng phải rebuild state song song → 2 tabs cạnh tranh CPU/Network → freeze.**
> 
> **Khi không nhấn F5: SPA chỉ thay route, $rootScope còn nguyên → không cần rebuild → tab mới hoạt động bình thường."**
