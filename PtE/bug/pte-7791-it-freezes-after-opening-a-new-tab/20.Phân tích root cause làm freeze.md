## Câu 2: Bỏ rAF — Tại sao trước cần mà giờ bỏ?

**Trước đây:** rAF cần vì init() kích hoạt heavy compilation (dropdowns, md-selects) → cần chờ dialog animation xong.

**Bây giờ:** Với `ng-if="initialized"`, init() chỉ làm:
1. `$scope.initialized = false` ← nhẹ
2. Push hours/minutes vào arrays ← nhẹ (array mutation, chưa có ng-repeat vì `ng-if=false`)
3. Gọi API `loadTaskGeneralData()` ← async, nhẹ

**Không có heavy work nào chạy TẠI THỜI ĐIỂM init()!** Heavy compilation chỉ xảy ra khi API response trả về → `initialized = true` → ng-if tạo DOM. Lúc đó dialog animation ĐÃ XONG, background bootstrap ĐÃ XONG.

→ rAF **không còn tác dụng gì** vì init() không cần timing trick nữa.

**Đơn giản: line 76-82 → thay bằng `init();`**
