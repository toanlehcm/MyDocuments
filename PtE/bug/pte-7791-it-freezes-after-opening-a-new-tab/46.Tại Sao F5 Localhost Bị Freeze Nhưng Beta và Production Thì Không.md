# 🔍 Tại Sao F5 Localhost Bị Freeze Nhưng Beta/Production Thì Không?

## TL;DR — Câu Trả Lời Ngắn

**Localhost load 478 file JS riêng lẻ qua 478 HTTP requests** → Browser xử lý script **tuần tự nhưng interleaved** → AngularJS bootstrap chạy **giữa lúc scripts chưa load xong** → animation callback throw error → `waitQueue` deadlock.

**Production gộp tất cả vào ~5-6 file JS bundle** → Load nhanh, đồng bộ, đúng thứ tự → Không có cửa sổ thời gian để race condition xảy ra.

---

## So Sánh Chi Tiết: Localhost vs Production

### 🏠 Localhost (`grunt serve`)

```
index.html
├── angular.js              ← file riêng (~1.3MB)
├── angular-animate.js      ← file riêng  
├── angular-touch.js        ← file riêng  
├── angular-material.js     ← file riêng  
├── ... (40+ bower files)
├── app.js                  ← file riêng  
├── ... (400+ app files)
└── safeAnimateAsyncRun.js  ← file riêng (cuối cùng)

📦 Tổng: ~478 HTTP requests
⏱️ Thời gian load: 3-8 giây (tùy máy)
```

### 🌐 Production (`grunt build`)

```
index.html    
├── vendor.js              ← 1 file (angular + animate + material + ...)
├── library.js             ← 1 file (dhtmlx + jquery-ui + ...)
├── scripts.js             ← 1 file (app.js + controllers + decorator)
└── template.js            ← 1 file (cached HTML templates)

📦 Tổng: ~5-6 HTTP requests
⏱️ Thời gian load: <1 giây (cached + CDN)
```

## 🎯 3 Nguyên Nhân Gốc Rễ: Tại Sao Localhost Bị

### Nguyên nhân 1: **Race Condition do "Waterfall Loading"** (70% khả năng)

```
LOCALHOST - 478 files, tuần tự:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
t=0ms     angular.js loaded → AngularJS core ready
t=50ms    angular-animate.js → $$animateAsyncRun registered
t=80ms    angular-material.js → $mdDialog ready
t=100ms   app.js → PtEMobile module created, .config() chạy
          ↓
          AngularJS bắt đầu bootstrap ngay lập tức
          ↓
t=150ms   Controller X loaded → trigger animation
t=200ms   Controller Y loaded → trigger animation
          ...
t=2000ms  safeAnimateAsyncRun.js loaded ← QUÁ MUỘN!
          Decorator chạy NHƯNG $$animateAsyncRun đã được inject
          vào các service từ trước đó rồi.
```

> [!CAUTION]
> **Đây là lý do decorator không work trên localhost!**  
> `$$animateAsyncRun` được inject vào `AnimateRunner` tại thời điểm `AnimateRunner` được tạo (line 6322 angular.js).  
> Decorator chỉ thay đổi service **SAU KHI** đã có instance cũ được inject rồi.

```
PRODUCTION - 5 files, đồng bộ:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
t=0ms     vendor.js loaded → TẤT CẢ angular loaded cùng lúc
t=50ms    scripts.js loaded → app.js + decorator + controllers
          ↓
          Decorator registered TRƯỚC khi bootstrap
          ↓
          $$animateAsyncRun đã được patch SẴN
          ↓ 
          Không có race condition!
```

### Nguyên nhân 2: **LiveReload Injection** (20% khả năng)

Localhost chạy `grunt serve` với LiveReload (port 35729):

```javascript
// Gruntfile.js line 87
livereload: 35729
```

LiveReload inject thêm JavaScript vào page. Khi F5:
1. Page unload → LiveReload socket disconnect
2. Page reload → LiveReload cố reconnect
3. Reconnect script chạy **giữa lúc** AngularJS đang bootstrap
4. Có thể trigger thêm DOM mutation → animation → stress test `waitQueue`

**Production KHÔNG có LiveReload** → không có sự can thiệp này.

---

### Nguyên nhân 3: **Cache Behavior Khác Nhau** (10% khả năng)

```html
<!-- index.html meta tags -->
<meta http-equiv="cache-control" content="max-age=0" />
<meta http-equiv="cache-control" content="no-cache" />
<meta http-equiv="expires" content="0" />
<meta http-equiv="pragma" content="no-cache" />
```

| Hành vi | Localhost | Production |
|---------|-----------|------------|
| F5 (Refresh) | 478 requests → all 304/200 | 5 requests → mostly 304 (cached) |
| Ctrl+F5 (Hard) | 478 requests → all 200 | 5 requests → all 200 |
| **Cache hit** | **Thấp** (no-cache headers) | **Cao** (CDN + filerev hash) |
| **Load time** | **3-8s** | **<1s** |

Mặc dù có `no-cache` meta tags, production vẫn nhanh hơn vì:
- `filerev` tạo hash vào filename → browser cache vĩnh viễn
- CDN cache (Cloudflare/S3)
- Ít request hơn = ít overhead hơn

## 🧪 Cách Kiểm Chứng

### Test 1: Chạy `grunt build` rồi serve từ `dist/`
```bash
grunt build
grunt serve:dist
```
Nếu **KHÔNG** bị freeze → Xác nhận nguyên nhân 1 (bundling giải quyết vấn đề).

### Test 2: Di chuyển decorator lên TRƯỚC app.js
Trong `index.html`, thay đổi vị trí `safeAnimateAsyncRun.js`:

```diff
<!-- TRƯỚC (hiện tại - DECORATOR Ở CUỐI) -->
- Line 382: app.js
- Line 803: safeAnimateAsyncRun.js (cuối, sau 400+ files)

<!-- SAU (đề xuất - DECORATOR NGAY SAU ANGULAR) -->
  Line 283: angular.js
  Line 285: angular-animate.js  
+ Line 286: safeAnimateAsyncRun.js  ← Di chuyển lên đây
  ...
  Line 382: app.js
```

> [!IMPORTANT]
> **NHƯNG** decorator dùng `angular.module('PtEMobile')` — module này chưa được tạo ở line 286!
> Vì `app.js` (line 382) mới tạo module `PtEMobile`.
> 
> **→ Cần refactor decorator để KHÔNG phụ thuộc vào PtEMobile module.**

## 💡 Giải Pháp Đề Xuất

### Option A: Đăng ký decorator vào module `ng` (thay vì `PtEMobile`)

```javascript
// safeAnimateAsyncRun.js — sửa lại
angular.module('ng').config(['$provide', function($provide) {
    $provide.decorator('$$animateAsyncRun', ['$delegate', '$$rAF', function($delegate, $$rAF) {
        // ... same logic ...
    }])
}])
```

**Ưu điểm:** `ng` module luôn tồn tại từ khi `angular.js` load → có thể đăng ký NGAY sau angular.js.

**Nhược điểm:** Patching core module, cần test kỹ.

### Option B: Tạo module riêng, inject vào trước

```javascript
// safeAnimateAsyncRun.js
angular.module('safeAnimate', []).config(['$provide', function($provide) {
    $provide.decorator('$$animateAsyncRun', ['$delegate', '$$rAF', function($delegate, $$rAF) {
        // ... same logic ...
    }])
}])
```

Rồi trong `app.js`:
```javascript
angular.module('PtEMobile', ['safeAnimate', 'ngAnimate', ...])
```

**Ưu điểm:** Clean separation, không patch core module.

**Nhược điểm:** Vẫn phụ thuộc thứ tự load file.

### Option C: IIFE tự thực thi — Patch trực tiếp `$$AnimateAsyncRunFactoryProvider`

```javascript
// safeAnimateAsyncRun.js — chạy NGAY khi load, không cần module
;(function() {
    // Monkey-patch angular.js internal TRƯỚC khi bootstrap
    // ... approach khác, patch ở level factory provider ...
})()
```

## 📊 Tóm Tắt

| Yếu tố | Localhost | Production |
|---------|-----------|------------|
| Số HTTP requests | **~478** | **~5-6** |
| Script loading | **Waterfall, async gaps** | **Bundled, synchronous** |
| Thời gian bootstrap | **3-8s** (nhiều gaps) | **<1s** (liên tục) |
| LiveReload | **Có** (can thiệp DOM) | **Không** |
| Cache | **Weak** (no-cache) | **Strong** (filerev + CDN) |
| Race condition window | **Rộng** (seconds) | **Gần 0** |
| Decorator timing | **Quá muộn** (file 478/478) | **Cùng lúc** (trong bundle) |

> [!IMPORTANT]
> **Kết luận chính:** Bug không phải là "localhost có mà production không có". Bug **LUÔN tồn tại** trong code — nhưng cửa sổ thời gian để trigger nó ở production **gần bằng 0** vì bundling loại bỏ race condition.
