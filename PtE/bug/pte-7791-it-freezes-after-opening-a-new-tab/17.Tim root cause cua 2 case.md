## Tóm tắt

**Root cause Case 2 KHÔNG phải performance hay animation.** Đó là **bug logic trong session validation**.

### Chuỗi bug:

1. Logout reset `applicationCount = 0` trong localStorage
2. F5 xóa sạch context
3. Login set lại `localStorage.LOGIN_IN_DATA` + `sessionStorage.LOGIN_IN_DATA`
4. User click "View associated task" → `window.open()` với `noopener` → new tab
5. New tab: `sessionStorage` = **TRỐNG** (vì `noopener` tạo isolated browsing context)
6. `applicationCount` = 0 → increment → 1
7. Logic check: `applicationCount(1) <= 1 && !sessionData(null)` = **TRUE**
8. → Gọi `clearDataSession()` **NGAY TRONG controller init** → redirect home → **state corruption/vòng lặp**

### Fix đề xuất

Thêm `&& !userData` vào condition ở `sessionFactory.js:1535`:
```diff
- if(applicationCount <= 1 && !sessionData){
+ if(applicationCount <= 1 && !sessionData && !userData){
```

Nghĩa là: chỉ kick về login khi CẢ `sessionStorage` VÀ `localStorage` đều trống (user thực sự chưa login).

Xin anh xác nhận 3 câu hỏi trong phần "Open Questions" của artifact để em apply fix.
