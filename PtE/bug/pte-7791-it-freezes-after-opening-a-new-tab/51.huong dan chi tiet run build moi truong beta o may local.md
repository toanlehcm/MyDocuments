### Lỗi 1: Truy cập `http://0.0.0.0:9002` báo `ERR_ADDRESS_INVALID`

- **Nguyên nhân cốt lõi:**
  Địa chỉ `0.0.0.0` trong mạng máy tính không phải là một địa chỉ IP thật trỏ tới một cái máy nào. Nó là một "Meta-address" (địa chỉ giả). Khi cấu hình server NodeJS khởi chạy ở `0.0.0.0`, nghĩa là ta đang bảo server: *"Hãy lắng nghe mọi yêu cầu gửi đến từ TẤT CẢ các card mạng hiện có (cả localhost và mạng LAN)"*. 
  Tuy nhiên, khi anh nhập cục diện `0.0.0.0` vào thanh URL của Chrome (trên Windows), Chrome sẽ chặn lại và báo địa chỉ không hợp lệ vì trình duyệt bắt buộc phải gửi yêu cầu đến một địa chỉ đích cụ thể (như `localhost` hoặc `127.0.0.1` hoặc IP mạng LAN như `192.168...`).

- **Cách giải quyết:** 
  Tuyệt đối không gõ `0.0.0.0` lên trình duyệt. Anh luôn dùng `http://localhost:9002` hoặc `http://127.0.0.1:9002`.

---

### Lỗi 2: Nhấn F5 ở `http://localhost:9002/patient_dashboard` báo `Cannot GET`

- **Nguyên nhân cốt lõi:**
  PtEverywhere là ứng dụng Single Page Application (SPA), dùng `html5Mode` cho router.
  - Khi anh đang ở `localhost:9002` và bấm link trong app để tới được `patient_dashboard`, Javascript tự "giả lập" đổi URL trên trình duyệt mà KHÔNG hề gửi yêu cầu lên server.
  - Nhưng khi anh **bấm F5**, trình duyệt thực sự gửi một lệnh `GET` trực tiếp hỏi thư mục gốc (server grunt): *"Cho tôi xin file patient_dashboard/index.html"*
  - Vấn đề nằm ở file **`Gruntfile.js`** của project này. Em vừa xem code thì thấy: môi trường Dev (`grunt serve`) đã được cài sẵn cơ chế `modRewrite` (Tự động chuyển mọi lỗi báo thiếu file về lại `index.html` cho Angular tự xử lý tiêp). Tuy nhiên, môi trường Build (`grunt serve:dist`) **VÔ TÌNH BỊ THIẾU** đoạn code cấu hình `modRewrite` đó. Đây là server file tĩnh đơn thuần, nó tìm không thấy file thì nó báo `Cannot GET`. Cụ thể:

```javascript
// Gruntfile.js (line 90-109) - Môi trường Dev (Có Setup Fallback)
livereload: {
  options: {
    middleware: function (connect) { return [ modRewrite([...]) ... ] } // <--- CÓ REWRITE
  }
}

// Gruntfile.js (line 126-131) - Môi trường đóng gói Dist (Quên setup)
dist: {
  options: {
     open: true,
     base: '<%= yeoman.dist %>' // <--- KHÔNG CÓ REWRITE chặn URL
  }
}
```

- **Cách giải quyết để Test ngay lập tức (Không cần sửa Gruntfile):**
  1. Anh vào lại trang chủ `http://localhost:9002` bằng F5 (hoặc gõ `/`).
  2. Dùng chuột click để điều hướng vào `Tasks` hoặc `Patient`.
  3. Để tái hiện lại trường hợp Freeze, **anh không được bấm F5** trên trình duyệt (vì bị lỗi ngắt URL như trên). Thay vào đó, **anh hãy F5 trang chủ (nếu cần), rồi mở các popup Add Task liên tục** hoặc bấm các nút kích hoạt animation nhiều lần.
  4. Test trên môi trường này cốt yếu là để chắc chắn file `scripts.js` chứa đoạn decorator try catch và mọi thứ logic Dialog/Show-Hide chạy bình thường 100%.
