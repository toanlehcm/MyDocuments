## 🔬 PHÂN TÍCH ROOT CAUSE TRIỆT ĐỂ

### Dòng chảy chính xác khi freeze xảy ra

```
viewAssociatedTasks(patientId, 'Appointment', appointmentId)
  → commonService.openInternalLink(`/tasks/${patientId}/Appointment/${id}`, true)
    → $window.open(url, '_blank', 'noopener,noreferrer')
      → MỞ TAB MỚI: /tasks/xxx/Appointment/yyy
```

Trong **TAB MỚI** này, Angular bootstrap toàn bộ app từ đầu:

```
1. sessionFactory() → API call v2getMySession (chờ response)
2. Response → populate $rootScope (clientData, permissions, ...)
3. → TasksManagementCtrl.init()
4.   → Promise.all([loadTaskUserPreferences(), loadTaskGeneralData()])
5.     → 2 API calls đồng thời
6.     → Response về → initFilterDefault() → parseTasks
7.     → $scope.filterModel được update → digest storm
8.   → applyFilter() → loadTasks() → API call thứ 3
9.     → Response → parseTasks → THÊM digest cycles

TAT CẢ TRÊN đang chạy ĐỒNG THỜI với:
- Route change animation ($animate)
- Template fetching (tasksManagementPtE.html)
- Compiler creating dropdown directives
- Hàng chục $watch trong các components
```

### Tại sao trước khi fix (không có noopener) → freeze ở tab 7-8?

**Trước**: Tất cả tab shared 1 process. Tab 1-6 mỗi tab "đóng góp" thêm watchers + memory pressure vào process chung. Đến tab 7-8, process quá tải → rAF bị starved → `$animate.enter()` deadlock.

**Cơ chế**: Background tab throttling + Shared process overhead → Tích lũy dần → Bùng nổ ở tab 7-8.

### Tại sao sau khi thêm noopener → freeze NGAY sau logout+login?

**Vì `noopener,noreferrer` đã loại bỏ triệu chứng shared process, nhưng LỘ RA root cause thực sự:** Tab mới bootstrap từ đầu, main thread bị overwhelm bởi tất cả cùng lúc.

Sau logout + F5 + login, điều quan trọng xảy ra:

| Thứ | Trạng thái | Ảnh hưởng |
|-----|-----------|-----------|
| **`$templateCache`** | **TRỐNG** (in-memory, bị xóa khi F5) | Template phải HTTP fetch → async delay |
| **V8 Code Cache** | Có thể warm hoặc invalidated | JIT compilation nặng hơn lần đầu |
| **HTTP Cache (browser)** | **Có thể warm** | Nhưng vẫn phải qua async flow |
| **`sessionFactory` cache** | **TRỐNG** (in-memory) | Session API call bắt buộc |
| **localStorage** | **CÒN NGUYÊN** | `LOGIN_IN_DATA`, user preferences → KHÔNG phải vấn đề |
| **sessionStorage** | **CÒN NGUYÊN** (F5 không xóa) | Token session sống sót |
| **`$rootScope` watchers** | **RESET** nhưng tái tạo nhanh | Mỗi controller init tạo lại watchers |

> **Kết luận**: localStorage/sessionStorage/cookie KHÔNG phải nguyên nhân. Vấn đề là **tất cả initialization work chạy cùng 1 frame khi tab mới mở**, tranh chấp rAF với `$animate`.

### Tại sao lần thứ 2 trở đi không freeze?

Sau lần mở đầu tiên (dù có freeze):

1. **`$templateCache`** đã có template → lần 2 là sync, không async
2. **V8** đã JIT compile code → runs nhanh hơn 5-10x
3. **API response** có thể vẫn fresh trong HTTP cache → nhanh hơn
4. **Angular compile cache** đã cached link functions cho dropdown directives

→ Tổng thời gian init giảm từ **~800ms xuống ~100ms** → rAF không bị starved → KHÔNG freeze.

## ✅ SOLUTION ỔN ĐỊNH NHẤT: Deferred Rendering

**Nguyên tắc**: Tách dialog thành 2 phase — mở khung nhẹ trước, render nội dung nặng sau.

**Ưu điểm**:
- Không race condition (khác pre-fetch template)
- Không side effect (khác `$animate.enabled(false)`)
- Không phụ thuộc cache state, login state, hay tab count
- Hoạt động 100% trên mọi scenario: cold start, warm start, post-logout, incognito

**Cách implement** (2 file, thay đổi tối thiểu):

**File 1** — `popupAddEditTaskPtE.js` (controller):
```javascript
// Thêm đầu init()
$scope.dialogReady = false;

// Sau khi dialog DOM ổn định, bật nội dung nặng
requestAnimationFrame(function() {
    requestAnimationFrame(function() {
        $scope.$apply(function() {
            $scope.dialogReady = true;
        });
    });
});
```

**File 2** — `popupAddEditTaskPtE.html` (template):
```html
<!-- Wrap phần nặng (4 dropdown + editor) trong ng-if -->
<div ng-if="dialogReady">
    <!-- Patient Name, Related To, Assigned To, Notified To, CKEditor -->
</div>
<div ng-if="!dialogReady" style="text-align: center; padding: 40px;">
    <md-progress-circular md-mode="indeterminate"></md-progress-circular>
</div>
```

Kết quả: Dialog mở NGAY LẬP TỨC (khung + loading spinner), animation hoàn tất mượt mà, sau đó mới render nội dung nặng. **Không bao giờ freeze nữa.**
