Đoạn code trên **CHƯA ĐẠT CHUẨN HOÀN TOÀN** theo `.agent/quality-gates/skill-null-safety.md` và có thể sinh ra lỗi tiềm ẩn (Null Exception / Data inconsistency). 

Dưới đây là phân tích chi tiết và cách khắc phục:

### 🔍 Phân tích các rủi ro (Risk Analysis):

1. **Rủi ro Null/Undefined Exception trong vòng lặp (`TypeError: Cannot read property 'Name' of undefined/null`)**:
   - Bên trong hàm callback của `_.find`: `return c.Name === ...` và `return s.Name === ...`.
   - Nếu trong mảng `$scope.countries` hoặc `$scope.states` vô tình có chứa một phần tử `null` hoặc `undefined` (do API trả về lỗi hoặc mảng bị thay đổi), việc gọi thẳng `c.Name` hay `s.Name` sẽ lập tức làm crash toàn bộ JS (trắng trang/đứng nút Save).
   - **Cách fix:** Cần thêm Guard Pattern: `return c && c.Name === ...`. Hoặc dùng Lodash shorthand `_.find($scope.states, { Name: ... })` (Lodash tự handle null-safe ngầm).

2. **Rủi ro dữ liệu rác (Data Inconsistency) cho trường hợp Country**:
   - Code hiện tại: `if ($scope.countries && locationData.AddressDetails.Country) { ... }`
   - Nếu user xóa trắng Country trên UI (lúc này `locationData.AddressDetails.Country` = `""` hoặc `null`), block `if` này sẽ bị bỏ qua.
   - Hậu quả: `CountryId` bị kẹt lại ID cũ trong khi `Country` text đã bị xóa trắng. Gửi lên server vẫn lưu `CountryId` cũ.
   - **Cách fix:** Cần thêm block `else { locationData.AddressDetails.CountryId = null; }` (Tương tự như cách đã làm với State).

3. **Rủi ro tương tự cho trường hợp State bị xóa trắng**:
   - Nếu user xóa trắng State (value = `""`), đoạn `if ($scope.states)` vẫn chạy, nó sẽ thực hiện loop để tìm 1 state có `Name == ""`. Tuy logic cuối cùng vẫn trả về `null` (vì không tìm thấy state nào tên rỗng), nhưng nó làm lãng phí performance.
   - **Cách fix:** Gộp chung điều kiện `if ($scope.states && locationData.AddressDetails.State)`.

        // Sync CountryId/StateId from directive-managed text values before save.
