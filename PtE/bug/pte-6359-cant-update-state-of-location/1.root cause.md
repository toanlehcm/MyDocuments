## 🐛 Root Cause Analysis — PTE-6359: State không update được

Tôi đã trace qua toàn bộ flow và tìm ra **3 tầng** của bug này:

---

### Tầng 1: Directive chỉ update `.State`, không update `.StateId`

**File:** `countryStateCityPtE.js`

```js
// Line 119: khi user gõ text
$scope.ngModel = text;  // chỉ update ngModel = AddressDetails.State (text)

// Line 126: khi user chọn item từ dropdown  
$scope.ngModel = item.text;  // vẫn chỉ update .State text
```

Directive `country-state-city-pte` **KHÔNG BAO GIỜ** cập nhật `StateId`. Nó chỉ biết `field-type="state"` và bind vào `ng-model="currentEditLocation.AddressDetails.State"`.

---

### Tầng 2: `saveLocation()` cố gắng resolve State từ StateId — và overwrite lại

**File:** `locationManagementATP.js` — **line 1042–1047**:

```js
for (var j in $scope.states) {
    if ($scope.states[j]._id === locationData.AddressDetails.StateId) {
        locationData.AddressDetails.State = $scope.states[j].Name;  // ← GHI ĐÈ!
        break;
    }
}
```

Vì `StateId` **chưa bao giờ được directive cập nhật**, nó vẫn giữ **ObjectId cũ từ DB** (ví dụ `AK-583fd89d4a94540574d81e1a`). Vòng lặp này tìm thấy match với state cũ và **ghi đè** `.State` user vừa sửa → revert về giá trị cũ.

---

### Tầng 3: Bad data trong DB (bonus issue)

Screenshot của bạn cho thấy `StateId` đang chứa giá trị như `AK-583fd89d4a94540574d81e1a` — là một **ObjectId ghép tên state**. Đây là legacy data format không chuẩn.
