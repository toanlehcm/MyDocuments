| Vấn đề | Trước | Sau |
|---|---|---|
| `locationData` có thể null | `if (locationData.AddressDetails)` → crash | `if (locationData && locationData.AddressDetails)` ✅ |

### #2 — State validation `error = true` (đúng, không cần sửa)

Logic anh tự thêm **đúng hoàn toàn** và chuẩn convention:

```
if (State có giá trị) → validate format → set invalidState
else (State rỗng)    → StateErr = true, error = true  ✅ chặn submit
```

> ⚠️ **Lưu ý quan trọng:** Block validation State nằm bên trong `if (locationData.AddressDetails && locationData.AddressDetails.Country)` (line 844). Điều này có nghĩa: nếu **Country rỗng**, State sẽ không được validate. Đây là design intent hiện tại (Country là điều kiện tiên quyết). Nếu business logic yêu cầu phải luôn validate State dù Country rỗng thì cần tách ra riêng.
