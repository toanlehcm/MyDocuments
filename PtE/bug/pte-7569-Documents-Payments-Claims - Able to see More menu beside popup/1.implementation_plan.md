# PTE-7569: More Menu Persists Beside Popup

Bug: Khi chọn More > Fax/Print/Preview trong Documents/Payments/Claims tab, "More" menu vẫn hiển thị bên cạnh popup.

### 🎯 ROOT CAUSE (Documents Tab - Primary)

> [!CAUTION]
> **Bằng chứng quan trọng nhất:** Hàm `faxPDF` trong [documentsATP.js](file:///d:/Sources/pteverywhere/Client/app/scripts/controllers/patientDashboard/documentsATP.js#L3665-L3667) có dòng:
> ```javascript
> $scope.faxPDF = function (data, ev) {
>     $scope.currentItem = data;
>     $('#btn-more').click(); // ← Chỉ có faxPDF có dòng này!
>     getFilePDFDocument(data._id, 'VIEW_PDF').then(function (dataResponse) { ... })
> ```
> Dòng `$('#btn-more').click()` thủ công trigger click lên toggle button để đóng dropdown. Nhưng `printPDF`, `viewPDF`, `downloadPDF` thì **KHÔNG** có dòng này.

**Kết luận:** Bootstrap dropdown tự động close khi click item **trong trường hợp bình thường**. Nhưng CSS class `dropdown-menu__slow-hidden` ghi đè `display: block` vĩnh viễn lên dropdown menu, khiến mechanism đóng của Bootstrap bị vô hiệu hóa một phần.

#### CSS `dropdown-menu__slow-hidden` ([_utilities.scss](file:///d:/Sources/pteverywhere/Client/app/styles/base/_utilities.scss#L972-L999)):

```scss
// LUÔN display: block — ghi đè Bootstrap's display: none
.dropdown-menu__slow-hidden {
  max-width: 0;
  max-height: 0;
  display: block;       // ← BUG ENABLER: Luôn block, không bao giờ none
  overflow: hidden;
  transition: max-height 0.3s ease-in, max-width 0.3s ease-in, opacity 0.1s ease-in;
  opacity: 0%;
}

// Khi parent có .show → menu hiện ra
.show>.dropdown-menu__slow-hidden {
  max-width: 100vw;
  max-height: 100vh;
  opacity: 100%;
}
```

**Chuỗi sự kiện gây bug:**

1. User click "More" → Bootstrap adds `.show` to parent → menu visible (opacity 100%)
2. User click "Print" / "Preview" / "Fax" → `ng-click` fires the action function
3. Bootstrap's document click handler **should** remove `.show` → trigger CSS transition → menu fades away
4. **NHƯNG:** Khi action mở `$mdDialog` hoặc `printJS`, Angular Material's `$mdDialog` tạo backdrop overlay trùm toàn bộ document. Overlay này **chặn event bubbling** lên document, khiến Bootstrap không thể detect "click outside" để đóng dropdown
5. `printJS({showModal: true})` cũng tạo modal overlay và gọi `window.print()` sẽ **block main thread**, freeze CSS transition
6. Kết quả: `.show` class **không bị remove** → dropdown menu vẫn visible 100%

> [!IMPORTANT]
> **Tại sao `faxPDF` có `$('#btn-more').click()`?** Vì dev trước đã nhận ra bug này cho fax và thêm workaround bằng cách click toggle button trước khi mở dialog. Nhưng quên áp dụng cho `printPDF`, `viewPDF`, `downloadPDF`.

## Proposed Solutions (Trade-off Analysis)

### Solution A: Quick Fix — Đóng dropdown trước khi mở popup ⭐ Recommended

**Ý tưởng:** Thêm logic đóng Bootstrap dropdown / `md-menu` **trước** khi mở `$mdDialog` hoặc `printJS`.

**Documents tab** — Cần fix các hàm trong [documentsATP.js](file:///d:/Sources/pteverywhere/Client/app/scripts/controllers/patientDashboard/documentsATP.js):

```javascript
// Helper function — đóng dropdown bằng cách remove .show class
function _closeMoreDropdown() {
  // Tương tự workaround đã có ở faxPDF nhưng dùng Bootstrap API chuẩn
  $('.dropdown.show .dropdown-toggle, .dropdown.show [data-toggle="dropdown"]')
    .dropdown('hide')
}
```

Thêm `_closeMoreDropdown()` vào đầu: `printPDF`, `viewPDF`, `downloadPDF` (faxPDF đã có workaround riêng).

**Payments tab** — Tương tự trong [paymentPatientATP.js](file:///d:/Sources/pteverywhere/Client/app/scripts/controllers/patientDashboard/paymentPatientATP.js):
Thêm đóng dropdown vào `printInvoiceInfo`, `viewPDFInvoiceInfo`, `faxPDF`, `downloadInvoiceInfo`, `sendEmail`.

**Claims tab** — Nếu `md-menu` cũng bị:
`$mdMenu.close()` trước khi gọi action (nhưng cần verify trước).

| Aspect | Rating |
|--------|--------|
| **Effort** | ⭐ Thấp — Thêm 1 dòng code mỗi hàm |
| **Risk** | ⭐ Thấp — Chỉ thêm close, không đổi logic |
| **UX Impact** | ✅ Menu biến mất trước khi popup hiện |
| **Consistency** | ⚠️ Phải nhớ thêm cho mỗi action mới |

---

### Solution B: CSS Fix — Dùng z-index để ẩn dropdown khi `$mdDialog` active

**Ý tưởng:** Khi `$mdDialog` backdrop active, force ẩn tất cả dropdown:

```scss
// Khi md-dialog backdrop hiện, ẩn hết dropdown
.md-dialog-is-showing .dropdown-menu__slow-hidden,
.md-dialog-is-showing .dropdown-menu.show {
  opacity: 0 !important;
  pointer-events: none !important;
  visibility: hidden !important;
}
```

| Aspect | Rating |
|--------|--------|
| **Effort** | ⭐ Rất thấp — 1 rule CSS |
| **Risk** | ⚠️ Trung bình — Có thể ảnh hưởng dropdown ở nơi khác nếu cùng page có dialog |
| **UX Impact** | ✅ Tự động ẩn mọi dropdown khi dialog mở |
| **Consistency** | ✅ Áp dụng global, không cần sửa JS |

---

### Solution C: Refactor — Bỏ `dropdown-menu__slow-hidden`, dùng Bootstrap animation chuẩn

**Ý tưởng:** Loại bỏ CSS custom `dropdown-menu__slow-hidden` và thay bằng Bootstrap's built-in dropdown animation hoặc `md-menu` thống nhất.

| Aspect | Rating |
|--------|--------|
| **Effort** | 🔴 Cao — Phải test tất cả dropdown dùng class này (Documents, Contact Logs, Profile) |
| **Risk** | 🔴 Cao — Thay đổi animation behavior ở nhiều nơi |
| **UX Impact** | ⚠️ Mất hiệu ứng "slow-hidden" mà UX team có thể đã design |
| **Consistency** | ✅ Sử dụng chuẩn Bootstrap |
