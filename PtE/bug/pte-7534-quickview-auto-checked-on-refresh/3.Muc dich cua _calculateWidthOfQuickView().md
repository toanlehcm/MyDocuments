## `_calculateWidthOfQuickView` làm gì?

```javascript
function _calculateWidthOfQuickView(timeout) {
  setTimeout(() => {
    $("#container-tag ul li.tag").each(function () {
      $(this).css("width", "");            // reset width về auto
      $(this).css("width", $(this).outerWidth() + "px"); // gán lại width tính từ DOM
    });
  }, timeout || 500);
}
```

**Mục đích:** Đo lại width thực tế của từng tag trong Quick View list (dùng `pte-sortable` drag-drop). Sau khi list thay đổi (thêm/bớt tag), các tag cần biết kích thước chính xác để drag-drop hoạt động đúng.

**Quan trọng:** Function này **luôn delay 500ms** (`setTimeout`). Nghĩa là mọi lần gọi đều async, không block UI.

## Gọi nhiều lần có an toàn không?

**Hoàn toàn an toàn, vì:**

1. **Pure DOM operation** — chỉ đọc và set `width`, không thay đổi state AngularJS, không có side effect
2. **Async via `setTimeout`** — các lần gọi cách nhau > 500ms (API round-trip thường 200-500ms), nên không overlap
3. **Idempotent** — gọi 10 lần cũng cho kết quả như gọi 1 lần: tag có đúng width thực tế
