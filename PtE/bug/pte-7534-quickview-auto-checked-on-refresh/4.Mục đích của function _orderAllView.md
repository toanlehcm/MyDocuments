Mục đích của function `_orderAllView` là để **sắp xếp thứ tự hiển thị của danh sách các view trong dropdown "+ Add more view"**. 

Logic sắp xếp chia danh sách thành 2 nhóm, hiển thị từ trên xuống dưới như sau:

**1. Nhóm Đã Chọn (Selected Views) - Nằm ở trên cùng:**
```javascript
const viewSelected = _.orderBy(
  allViewList.filter((v) => v.selected),
  ["Order"], // Sắp xếp theo thuộc tính Order
  ["asc"]    // Tăng dần
);
```
- Lọc ra các view đang được check (`v.selected === true`).
- Sắp xếp nhóm này theo trường `Order` (tăng dần). Đây chính là thứ tự tương ứng với các tag đang hiển thị trên thanh Quick View (người dùng có thể kéo thả để đổi `Order` này).

**2. Nhóm Chưa Chọn (Unselected Views) - Nằm ở dưới:**
```javascript
const viewUnselected = allViewList
  .filter((v) => !v.selected)
  .sort(function (a, b) {
    return moment(a.CreatedAt).valueOf() > moment(b.CreatedAt).valueOf() ? -1 : 1;
  });
```
- Lọc ra các view chưa được check (`v.selected === false`).
- Sắp xếp nhóm này theo **thời gian tạo (`CreatedAt`) mới nhất xếp trước** (giảm dần / descending).

**Kết quả (`return [...viewSelected, ...viewUnselected]`):**
Khi mở dropdown "+ Add more view", bạn sẽ luôn thấy:
- Các view **đang được tick** nằm gom lại ở **trên cùng**, thứ tự y hệt trên thanh tab.
- Các view **chưa được tick** nằm ở **dưới**, cái nào mới tạo sẽ nằm trên. 

Đây là một UX pattern rất tốt giúp user dễ dàng tìm thấy các view họ đang dùng hoặc vừa mới tạo.
