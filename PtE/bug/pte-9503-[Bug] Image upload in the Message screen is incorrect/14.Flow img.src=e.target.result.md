### #2. img.onload có phải Promise không?

**Không phải Promise**, nhưng cơ chế tương tự — đó là **callback-based async** (event listener).

Thứ tự thực thi thực tế:

```javascript
// Line 74: ĐĂNG KÝ callback (chưa chạy, chỉ khai báo "khi nào load xong thì gọi")
img.onload = function() {   // ← chưa chạy
    // ... tất cả code xoay canvas ở đây
}

// Line 168: GÁN src → browser BẮT ĐẦU load ảnh (async)
img.src = e.target.result;  // ← trigger load

// ... browser load xong + tự xoay EXIF → fire img.onload callback
// → lúc này code trong img.onload mới chạy
```

**Anh hiểu đúng:** `img.src = ...` (line 168) chạy TRƯỚC → browser load + auto-xoay EXIF → `img.onload` (line 74) chạy SAU. Nên khi code trong `img.onload` thấy `img.width/height`, ảnh ĐÃ được browser xoay rồi. Rồi code lại `ctx.transform` xoay thêm → **double rotation**.
