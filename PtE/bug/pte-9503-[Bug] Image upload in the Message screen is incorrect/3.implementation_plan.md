# PTE-9503: Fix Image Upload Orientation — Message Screen

## Tóm tắt vấn đề

Ảnh portrait (chụp dọc) bị hiển thị sai thành landscape (nằm ngang) khi upload qua Message screen trên Samsung Galaxy S22/A24 (Android 16, Chrome WebView 146).

---

## 📍 Dòng code gây bug — Tóm gọn cho anh debug

### Luồng chạy:

```
HTML line 436-437 (messageATP.html):
  <input type="file" custom-on-change="chooseImage">
       ↓
Controller line 733-734 (messageATP.js):
  imageFactory.getOrientation(file, function(orientation) {        ← Đọc EXIF → trả về 6 (portrait)
      imageFactory.resetOrientation(file, orientation, null, ...)  ← BUG Ở ĐÂY
       ↓
Factory line 45-88 (imageFactory.js):
  img.onload → img.width/height ĐÃ bị browser auto-rotate      ← Browser Chrome 81+ tự xoay rồi
  canvas: ctx.transform(orientation=6) → xoay thêm 90° nữa     ← XOay LẦN 2 → sai
```

### Dòng gốc rễ:

| File | Line | Vấn đề |
|------|------|--------|
| `imageFactory.js` | **45–72** | `resetOrientation` — `img.onload` lấy `img.width/height` đã bị browser tự xoay, rồi lại `ctx.transform()` xoay thêm 1 lần nữa = **double rotation** |
| `messageATP.js` | **733–734** | Luôn gọi `getOrientation` → `resetOrientation` mà không check browser đã tự xoay chưa |

### Cơ chế kỹ thuật:

> Từ **Chrome 81+** (2020), tất cả Chromium-based browsers (bao gồm WebView trên Android) **tự động apply EXIF orientation** khi load ảnh vào `<img>` hoặc `new Image()`. Nghĩa là khi `img.onload` fires, `img.width` và `img.height` đã phản ánh **ảnh đã xoay đúng**.
>
> Nhưng `getOrientation` vẫn đọc raw EXIF bytes → trả `orientation = 6`, rồi `resetOrientation` lại canvas transform thêm 90° nữa → **portrait thành landscape**.

## Proposed Changes

### Option A: Detect browser auto-rotation (✅ Recommended)

Thêm một helper function vào `imageFactory.js` để **feature-detect** xem browser hiện tại có tự xoay EXIF hay không. Nếu có → skip transform, chỉ re-encode ảnh (giữ nguyên logic convert to Blob).

### Option B: Bỏ hoàn toàn getOrientation + resetOrientation, dùng canvas trực tiếp

Vì tất cả browsers hiện đại (2020+) đều auto-rotate EXIF, có thể đơn giản hóa luồng: chỉ cần `new Image()` → `canvas.drawImage()` → `canvas.toBlob()`. Không cần đọc EXIF hay transform gì cả.

> ⚠️ **Rủi ro**: Nếu có user dùng browser cổ (Chrome < 81, Safari cũ) thì ảnh sẽ bị sai hướng.

## Trade-off Analysis

| Tiêu chí | Option A (Detect) | Option B (Remove) |
|----------|--------------------|--------------------|
| **Backward compatible** | ✅ Hỗ trợ cả browser cũ lẫn mới | ❌ Chỉ browser mới |
| **Độ phức tạp** | Trung bình — thêm 1 helper function | Thấp — xóa code |
| **Risk regression** | Thấp — logic cũ vẫn chạy cho browser cũ | Cao — phá vỡ browser cũ |
| **Impact scope** | `imageFactory.js` thay đổi 1 file duy nhất | Phải sửa ~40+ caller sites |
| **Performance** | Tốt — detect chạy 1 lần, cache kết quả | Tốt — ít code hơn |
| **Maintainability** | Tốt — tập trung ở factory | Tốt — ít code |

> [!IMPORTANT]
> **Recommend Option A** vì:
> 1. PTE app chạy trên WebView (Android), nhưng cũng hỗ trợ browser trực tiếp — cần backward compatible
> 2. Chỉ sửa 1 file `imageFactory.js` — impact scope nhỏ nhất
> 3. `resetOrientation` được gọi ở **40+ chỗ** khắp codebase (avatar, exercise, insurance, message, clinic settings...) — fix tại factory sẽ fix cho tất cả
