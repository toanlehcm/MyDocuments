### 🎯 1. Mục đích của function `getOrientation`

**Nhiệm vụ chính:** Đọc các thông tin ẩn (metadata) bên trong 1 file ảnh (thường là định dạng JPEG/JPG) để tìm ra **hướng xoay gốc của ảnh** (Orientation tag nằm trong EXIF).

**Ví dụ đời thường:** Giống như khi anh cầm điện thoại lên chụp một tấm ảnh, máy ảnh không chỉ lưu lại hình ảnh mà còn kẹp thêm một cái "nhãn" ghi chú (gọi là EXIF) cho biết lúc đó điện thoại đang cầm dọc hay cầm ngang. Hàm này đóng vai trò như một người "kiểm kho", mở file ảnh ra, không thèm nhìn hình, chỉ lục tung các nhãn ghi chú lên để tìm đúng cái nhãn báo "hướng ảnh" để báo về, giúp hệ thống sau này biết cách tự động xoay ảnh cho chuẩn, không bị ngược hay nằm ngang.


```javascript
  var reader = new FileReader();
```
👉 Thuê một công cụ chuyên dụng của trình duyệt tên là `FileReader` để chuẩn bị đọc file.

```javascript
  reader.onload = function(e) {
    delete reader.onload;
```
👉 `onload`: Dặn công cụ đọc là "Đọc xong thì làm tiếp những việc dưới đây nhé".
👉 `delete reader.onload`: Đọc xong rồi thì xóa ngay cái lệnh này đi để dọn dẹp bộ nhớ (phòng ngừa memory leak theo chuẩn AngularJS/JS).

```javascript
    var view = new DataView(e.target.result);
```
👉 Chuyển dữ liệu vừa đọc được thành dạng `DataView`. Hãy tưởng tượng ảnh giờ không phải là bức tranh nữa, mà là một chuỗi mã Morse kéo dài vô tận, và `DataView` là cái kính lúp giúp ta săm soi từng điểm một (từng byte).

```javascript
    if (view.getUint16(0, false) != 0xFFD8) return callback(-2);
```
👉 Soi ngay 2 byte đầu tiên của file. Nếu nó không khớp với mã `0xFFD8` thì báo về `-2`. Đây là "chữ ký" bắt buộc của định dạng ảnh JPEG. Nếu không có mã này, chứng tỏ file không phải là file JPG chuẩn, nghỉ khỏe khỏi tìm mất công.

```javascript
    var length = view.byteLength, offset = 2;
    while (offset < length) {
```
👉 Lấy tổng chiều dài file. Đặt vị trí bắt đầu dò từ byte số 2 (do đã bỏ qua 2 byte chữ ký ở trên). Vòng lặp `while` bắt đầu chạy dọc theo file ảnh.

```javascript
      var marker = view.getUint16(offset, false);
      offset += 2;
```
👉 Đọc 2 byte tiếp theo để xem nó là cờ hiệu (marker) gì, đọc xong thì nhích kính lúp tiến tới 2 bước.

```javascript
      if (marker == 0xFFE1) {
```
👉 `0xFFE1` là lá cờ báo hiệu: "Khoan đã, đoạn này chứa thông tin Exif (chứa loại máy ảnh, ngày giờ, và **hướng ảnh** đây rồi)".

```javascript
        if (view.getUint32(offset += 2, false) != 0x45786966) return callback(-1);
```
👉 Đọc thử 4 byte tiếp theo. Mã `0x45786966` dịch ra mã ASCII chính là chữ **"Exif"**. Nếu không phải chữ này thì báo lỗi `-1` (không có thông tin Exif).

```javascript
        var little = view.getUint16(offset += 6, false) == 0x4949;
```
👉 Kiểm tra xem file này lưu byte theo chuẩn nào. Nếu là `0x4949` thì là chuẩn Little-Endian (thường của chip Intel). Đoạn này rất quan trọng để cách đọc số liệu phía sau không bị đảo ngược.

```javascript
        offset += view.getUint32(offset + 4, little);
        var tags = view.getUint16(offset, little);
        offset += 2;
```
👉 Nhảy cóc đến ngay khu vực chứa các thẻ (tags) thông tin, và đếm xem có tổng cộng bao nhiêu cái thẻ ở trong đó.

```javascript
        for (var i = 0; i < tags; i++)
          if (view.getUint16(offset + (i * 12), little) == 0x0112)
            return callback(view.getUint16(offset + (i * 12) + 8, little));
      }
```
👉 Bắt đầu dò từng cái thẻ một trong danh sách. 
👉 Mã `0x0112` là tiêu chuẩn quốc tế đại diện cho cái thẻ tên là **Orientation** (Hướng ảnh). 
👉 Nếu tìm trúng thẻ này, nhảy qua 8 byte để đọc giá trị của nó (VD: ảnh đang nằm ngang là số 6, bị lộn ngược là số 3, chuẩn là số 1) và lập tức báo lại kết quả qua `callback`.

```javascript
      else if ((marker & 0xFF00) != 0xFF00) break;
      else offset += view.getUint16(offset, false);
    }
```
👉 Lệnh `else` này dùng khi đoạn dữ liệu đang xét không phải là `0xFFE1` (không chứa Exif):
Nếu cờ không hợp lệ thì thoát luôn, còn nếu hợp lệ thì cứ đọc xem đoạn đấy dài bao nhiêu rồi **nhảy cóc** qua thẳng khúc đó cho nhanh, đỡ phải đọc từng chữ vô ích.

```javascript

```
