# Plan: Sử dụng cordova-plugin-androidx-adapter để fix Print trên Android 10+

## Bối cảnh

**Vấn đề:** `cordova-plugin-printer` (v0.7.3) không hoạt động trên Android 10+ vì:
1. **Build Error:** Plugin dùng `android.support.*` (cũ) trong khi Android 10+ yêu cầu `androidx.*` (mới)
2. **Runtime Crash:** Bug NullPointerException ở hàm `getPrintJob` trên Android 10 ([Issue #233](https://github.com/katzer/cordova-plugin-printer/issues/233))

**Đề xuất:** Cài `cordova-plugin-androidx-adapter` để fix lỗi #1 (build), sau đó patch code để fix lỗi #2 (runtime).

---

## Hiểu đúng vai trò của từng plugin

> [!IMPORTANT]
> `cordova-plugin-androidx-adapter` **KHÔNG PHẢI** plugin in ấn thay thế cho `cordova-plugin-printer`.
> Nó là **adapter build-time** — tự động đổi tên `android.support.*` → `androidx.*` trong Java code lúc build.

| Plugin | Vai trò | Link |
|---|---|---|
| `cordova-plugin-printer` v0.7.3 | Plugin in PDF/HTML trên thiết bị | [npm](https://www.npmjs.com/package/cordova-plugin-printer/v/0.7.3) |
| `cordova-plugin-androidx-adapter` | Adapter chuyển đổi Support Library → AndroidX lúc build | [npm](https://www.npmjs.com/package/cordova-plugin-androidx-adapter), [GitHub](https://github.com/nickhsharp/cordova-plugin-androidx-adapter) |

## Phân tích: Adapter có fix được vấn đề không?

### Vấn đề 1: Build Error (Lỗi biên dịch) — ✅ CÓ THỂ FIX

**Nguyên nhân:** `cordova-plugin-printer` import `android.support.v4.print.PrintHelper`, nhưng AndroidX đã đổi thành `androidx.print.PrintHelper`.

**Adapter fix được:** Vì nó tự động thay `android.support.*` → `androidx.*` trong toàn bộ Java source code lúc build.

**Bằng chứng:**
- [GitHub Issue #283](https://github.com/katzer/cordova-plugin-printer/issues/283): Nhiều dev xác nhận dùng adapter fix được lỗi build.
- [StackOverflow: AndroidXEnabled + adapter](https://stackoverflow.com/questions/56504522/how-to-fix-android-support-v4-import-error-in-cordova-plugin): Giải pháp được vote cao nhất.

### Vấn đề 2: Runtime Crash (Lỗi khi chạy) — ⚠️ KHÔNG FIX ĐƯỢC

**Nguyên nhân:** Hàm `getPrintJob` trong `PrintManager.java` trả về NULL trên Android 10, gây NullPointerException.

**Adapter KHÔNG fix được:** Vì đây là bug logic trong code Java của plugin, không liên quan đến tên thư viện.

**Bằng chứng:**
- [GitHub Issue #233](https://github.com/katzer/cordova-plugin-printer/issues/233): App crash trên Android 10 do NullPointerException.
- Workaround: Phải fork plugin và thêm null check thủ công.

## Proposed Changes (Nếu muốn thử)

### Phase 1: Cài adapter (fix build)

**Bước 1:** Thêm preference vào `config.xml`:
```xml
<platform name="android">
    <preference name="AndroidXEnabled" value="true" />
</platform>
```

**Bước 2:** Cài adapter:
```bash
cordova plugin add cordova-plugin-androidx-adapter
```

**Bước 3:** Clean build:
```bash
cordova platform rm android
cordova platform add android
cordova build android
```

### Phase 2: Patch runtime crash (fix NullPointerException)

Fork `cordova-plugin-printer` và sửa file `src/android/ext/PrintManager.java`:

Thêm null check trước khi gọi `getPrintJob`:
```java
// Thêm null check
if (method != null) {
    // perform invocation
}
```

> [!WARNING]
> Phase 2 yêu cầu fork repo, sửa Java code, tự maintain bản fork riêng — tăng tech debt.

## Đánh giá Trade-off

### ✅ Lợi ích
- Có khả năng build được app trên Android 10+
- Không cần migrate sang Capacitor (tiết kiệm effort)
- `cordova-plugin-androidx-adapter` là plugin phổ biến, ổn định (7M+ downloads)
- Không ảnh hưởng đến iOS (adapter chỉ tác động lên Android build)

### ❌ Đánh đổi & Rủi ro

| Rủi ro | Mức độ | Giải thích |
|---|---|---|
| Runtime crash trên Android 10 | 🔴 Cao | Adapter chỉ fix build, không fix bug logic. Cần patch thủ công. |
| Maintain fork riêng | 🟡 Trung bình | Nếu patch runtime bug, phải tự maintain bản fork của printer plugin |
| Adapter deprecated | 🟡 Trung bình | `cordova-android` 9+ đã built-in AndroidX, adapter có thể không cần nếu nâng cordova-android |
| Xung đột với plugin khác | 🟢 Thấp | Adapter chỉ đổi tên import, ít khi gây side-effect |
| Không test được đầy đủ | 🟡 Trung bình | Cần thiết bị Android 10+ thật để test runtime behavior |

cordova-plugin-androidx-adapter là một plugin Apache Cordova được thiết kế để tự động chuyển đổi các plugin Cordova cũ (sử dụng thư viện Android Support cũ) sang AndroidX, giúp ứng dụng tương thích với các phiên bản Android SDK mới hơn. Nó giải quyết các xung đột dependency khi xây dựng ứng dụng (build) mà không cần cập nhật thủ công từng plugin.
Các đặc điểm chính:
Tự động hóa: Tự động sửa lỗi xung đột giữa các thư viện android.support.* và androidx.* trong quá trình build.
Tương thích: Giúp các plugin Cordova không được cập nhật gần đây vẫn có thể chạy trên môi trường Android hiện đại (yêu cầu AndroidX).
Giải pháp tạm thời: Thường được sử dụng khi các plugin khác trong dự án chưa hỗ trợ chính thức AndroidX.
Nếu ứng dụng của bạn gặp lỗi Unable to find matching android.support hoặc các lỗi liên quan đến chuyển đổi thư viện, plugin này thường là giải pháp khắc phục.
