# Kế hoạch chi tiết: Cài đặt `cordova-plugin-androidx`

> [!CAUTION]
> **CẢNH BÁO ĐỎ TRƯỚC KHI BẮT ĐẦU:**
> `cordova-plugin-androidx` **KHÔNG PHẢI LÀ PLUGIN IN ẤN**. Nó không có khả năng nhận file PDF hay kết nối với máy in.
> Nếu anh xóa `cordova-plugin-printer` và thay bằng plugin này, **APP SẼ MẤT HOÀN TOÀN CHỨC NĂNG IN ẤN**. Nó chỉ là một công cụ hỗ trợ build code (dịch Support Library sang AndroidX), không phải "máy in mới".

Anh hãy đọc kỹ sự thật đánh đổi dưới đây trước khi quyết định.

---

## 1. Trả lời: Làm sao để kiểm tra thư viện Cordova version mấy?

Vì cấu trúc folder dự án PtE khá đặc biệt (các file nền tảng Android/iOS được đẩy thẳng vào `Client/app`), anh có thể kiểm tra version bằng 2 cách:

**Cách 1: Xem trực tiếp trong file code (Chính xác 100%)**
- Mở file `Client/app/cordova-ios/cordova.js` → Dòng 22 ghi: `var PLATFORM_VERSION_BUILD_LABEL = '6.0.0';` (Tức là `cordova-ios` bản 6.0.0).
- Mở file `Client/app/cordova-android/cordova.js` → Xem dòng ghi `PLATFORM_VERSION_BUILD_LABEL`.

**Cách 2: Dùng command line (Nếu máy có cài sẵn cordova toàn cục)**
Anh mở Terminal/CMD tại thư mục chứa file `config.xml` (thường là thư mục chứa source build mobile nguyên thủy, không phải `Client`) và gõ:
```bash
cordova platform ls   # Xem version của cordova-android và cordova-ios
cordova -v            # Xem version của cordova CLI
```

---

## 2. Hướng dẫn cài đặt và Config `cordova-plugin-androidx`

Nếu anh **vẫn bắt buộc phải cài** theo yêu cầu của cấp trên để thử nghiệm giải quyết lỗi build, đây là các bước:

### Bước 1: Gỡ bỏ plugin cũ (Nếu sếp yêu cầu "thay thế")
```bash
cordova plugin rm cordova-plugin-printer
```
> [!WARNING]
> Kể từ giây phút lệnh này chạy xong, toàn bộ code `$cordovaPrinter.print(url)` trong `commonService.js` sẽ bị **GÂY TRẮNG MÀN HÌNH HOẶC CRASH APP** vì plugin không còn tồn tại.

### Bước 2: Cài đặt plugin AndroidX
```bash
cordova plugin add cordova-plugin-androidx
```

### Bước 3: Đảm bảo config.xml cho phép AndroidX
Vào file `config.xml`, tìm thẻ `<platform name="android">` và thêm 2 dòng này:
```xml
<preference name="AndroidXEnabled" value="true" />
<preference name="GradlePluginKotlinEnabled" value="true" />
```

### Bước 4: Clean và Build lại
```bash
cordova platform rm android
cordova platform add android
cordova build android
```

## 3. Những gì thay đổi và Cái giá phải trả (Trade-offs)

Nếu áp dụng plan "Thay thế plugin" này, đây là bảng đánh giá:

| Đánh đổi | Tình trạng | Giải thích |
|---|---|---|
| **Khả năng Build Android 10+** | 🟢 Tốt hơn | Project sẽ tương thích hoàn toàn với AndroidX, không bị vướng class cũ. |
| **Tính năng IN ẤN (Print)** | 🔴 **CHẾT TOÀN TẬP** | Cả iOS và Android đều không in được nữa vì ta đã xóa tính năng in. |
| **Effort code lại (Code Rewrite)**| 🔴 Rất Cao | Team sẽ phải tìm một thư viện in ấn hoàn toàn mới (VD: của Capacitor) để đập đi xây lại luồng In. |

### Đề xuất của em (Hà - Logic Architect):
Anh hãy gửi file này cho Leader và nói: 
*"Sếp ơi, em đã tìm hiểu. `cordova-plugin-androidx` chỉ là **công cụ dịch mã nguồn**, nó không phải là **máy in**. Nếu mình dùng nó để thay thế `cordova-plugin-printer`, app sẽ build được nhưng **chắc chắn mất tính năng in**. Nếu muốn app in được trên Android 10+ thì chỉ có con đường duy nhất là tìm thư viện máy in mới (như Capacitor Printer) và mất vài tuần để code lại luồng in."*
