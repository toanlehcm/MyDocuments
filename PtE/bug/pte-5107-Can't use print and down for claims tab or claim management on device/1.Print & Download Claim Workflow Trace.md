# Print & Download Claim Workflow Trace

This document provides a detailed technical trace of the "Print" and "Download Claim" features in the PtEverywhere application, specifically within the Patient Dashboard (Claims tab).

## 📊 High-Level Sequence

```mermaid
sequenceDiagram
    participant UI as claimsATP.html
    participant CTRL as claimsPatientATP.js
    participant CS as commonService.js
    participant DS as downloadService.js
    participant VS as viewerService.js
    participant API as Backend API

    UI->>CTRL: announceClick(type, claim)
    CTRL->>CS: confirmPrintWithEob(claimId, type)
    CS-->>CTRL: { isPrintWithEob, isDownloadForFaxing }
    
    ALT type is 'print' or 'download'
        CTRL->>CS: getCMS1500FormUrl(claimId, isDownload)
        CS->>API: call('v4GetCMS1500FormUrl')
        API-->>CS: claimUrlPDF
        CS-->>CTRL: claimUrlPDF
    END

    CTRL->>CTRL: Switch (type)
    
    SECTION Print Flow
        CTRL->>CTRL: print1500(url, claimId)
        ALT Context: WEB
            CTRL->>UI: printJS({ printable: url })
            CTRL->>CS: _updateStatusAndPrintInfo(claimId)
        ELSE Context: APP (Mobile)
            CTRL->>UI: $cordovaPrinter.print(url)
        END
    END

    SECTION Download Flow
        CTRL->>CTRL: download1500(url, claim)
        ALT Context: WEB
            CTRL->>UI: window.URL.createObjectURL(Blob)
            CTRL->>UI: a.click() (trigger download)
        ELSE Context: APP (Mobile)
            CTRL->>DS: download(url, fileName)
            DS-->>CTRL: fileURI
            CTRL->>VS: showPDF(fileURI)
        END
        CTRL->>CS: updateStatusAndDownloadInfo(claimId)
    END
```

## 🛠️ Detailed Logic Trace

### 1. UI Entry Point
**File:** `d:\Sources\pteverywhere\Client\app\views\patientDashboard\claimsATP.html`

*   **Print Button:** 
    ```html
    <md-button ng-click="announceClick('print', claim)">Print</md-button>
    ```
*   **Download Button:**
    ```html
    <md-button ng-click="announceClick('download', claim)">Download CMS 1500</md-button>
    ```

### 2. Controller Handling
**File:** `d:\Sources\pteverywhere\Client\app\scripts\controllers\patientDashboard\claimsPatientATP.js`

*   **Function:** `announceClick(type, claim)` (Line ~3143)
    *   Calls `commonService.confirmPrintWithEob(claimId, type)` to determine if an EOB should be included.
    *   Calls `commonService.getCMS1500FormUrl(claimId, isDownload)` (Line ~3162) to fetch the PDF URL from the server.
    *   Upon resolution, handles the logic based on `type`.

### 3. Service Layer
**File:** `d:\Sources\pteverywhere\Client\app\scripts\service\commonService.js`

*   **Function:** `getCMS1500FormUrl(claimId, isDownload, isDownloadForFaxing)` (Line ~475)
    *   Calls API: `v4GetCMS1500FormURL`.
*   **Function:** `updateStatusAndDownloadInfo(claimId)` (Line ~592)
    *   Calls API: `v4UpdateDownloadStatus`.

### 4. Print Execution (`print1500`)
**File:** `d:\Sources\pteverywhere\Client\app\scripts\controllers\patientDashboard\claimsPatientATP.js`

*   **Logic:**
    *   **WEB:** Uses the `print-js` library: `printJS({ printable: url, type: 'pdf', showModal: true })`.
    *   **APP (Cordova):** Uses the printer plugin: `$cordovaPrinter.print(url)`.

### 5. Download Execution (`download1500`)
**File:** `d:\Sources\pteverywhere\Client\app\scripts\controllers\patientDashboard\claimsPatientATP.js`

*   **Logic:**
    *   **WEB:**
        1. Uses `pdfjsLib.getDocument(url)` to fetch the PDF data.
        2. Creates a Blob: `new Blob([data], {type: 'application/pdf'})`.
        3. Creates an Object URL: `window.URL.createObjectURL(...)`.
        4. Triggers a programmatic click on a hidden `<a>` tag with the `download` attribute.
    *   **APP (Cordova):**
        1. Calls `downloadService.download(url, fileName)`.
        2. Once saved locally, calls `viewerService.showPDF(fileURI)` to open the file.

## 📁 Related Factory Logic

### Download Service
**File:** `d:\Sources\pteverywhere\Client\app\scripts\factory\downloadService.js`
*   Handles directory creation and file transfer using Cordova's `File` and `FileTransfer` plugins.

### Viewer Service
**File:** `d:\Sources\pteverywhere\Client\app\scripts\factory\viewerService.js`
*   Uses `window.cordova.plugins.FileOpener2` to open the downloaded PDF in the system's default viewer.
