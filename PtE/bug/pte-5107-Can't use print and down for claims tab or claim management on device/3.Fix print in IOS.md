## 🔍 Root Cause: `dataResponse` là biến "ma" — không tồn tại

### Bằng chứng

`dataResponse` **không được khai báo ở bất kỳ đâu** trong file `claimsPatientATP.js`. Tìm kiếm toàn bộ file → 0 kết quả.

Đây là **copy-paste bug** từ `treamentNoteATP.js`, nơi `dataResponse` là tên biến dùng trong callback của `getFilePDFTreatmentNote().then(function(dataResponse){...})`. Dev đã copy cách dùng `$cordovaPrinter.print(dataResponse)` nhưng **quên thay** `dataResponse` bằng biến đúng là `url`.

### Phân tích code hiện tại

```js
function print1500(url, claimId) {
    if ($rootScope.appType === AppTypeConst.WEB) {
        // ✅ Đúng: dùng `url`
        printJS({ printable: url, type: 'pdf', showModal: true })
    } else {
        // ❌ Sai: `dataResponse` không tồn tại → undefined → print không có gì
        $cordovaPrinter.print(dataResponse);
    }
}
```

### ⚠️ Lưu ý Regression

Cần test kỹ trên iOS vì `$cordovaPrinter.print(url)` nhận vào URL của PDF từ server. Cần đảm bảo plugin `$cordovaPrinter` trên iOS có thể handle được URL dạng `http://...` (remote URL) hay chỉ nhận local file URI. Nếu plugin yêu cầu local file, thì cần refactor thêm bước download trước like flow của `download1500`.
