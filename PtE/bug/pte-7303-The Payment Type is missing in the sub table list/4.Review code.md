### 1. Tại sao lúc join thì gán là `"as" : "RemitPayment"`, nhưng khi dùng thì `$RemitPayment`?
Đây là quy ước cú pháp (Syntax Pattern) cốt lõi của **MongoDB Aggregation Framework**:
- `"as" : "RemitPayment"`: Ở stage `$lookup`, string này đóng vai trò là **Định danh (Identifier / Field Name Mới)**. Nó yêu cầu MongoDB khởi tạo một thuộc tính mới mang tên `RemitPayment` trên mỗi Document. Do đó, nó chỉ là một string thuần (literal string).
- `"$RemitPayment.Type"`: Khi ở các stage tiếp theo (như `$project`, `$addFields`), tiền tố `$` (Dollar Prefix) báo cho Engine nội bộ của MongoDB biết rằng đây là một **Truy xuất Giá trị (Variable Value Access/Field Path)** báo cho nó lấy giá trị của field `RemitPayment` thay vì truyền vào một chuỗi chữ "RemitPayment" cứng.

### 2. Code thay đổi ở Server và Client đã đúng Rules chưa? Có bị Null Exception không?
**Đánh giá tổng quan: Mã nguồn hoàn toàn HỢP LỆ và AN TOÀN (Thread-safe/Null-safe). Không xảy ra Null Exception.**

Em đã kiểm tra luồng Lifecycle và phân tích nguyên nhân tại sao nó an toàn:

**Ở tầng Server (`v2_ReportDAL.js`):**
- Ngay sau lệnh `$lookup`, Pipeline có sử dụng `$unwind`:
  ```javascript
  { 
      "$unwind" : {
          "path" : "$RemitPayment", 
          "preserveNullAndEmptyArrays" : false // <-- ĐÂY LÀ CHÌA KHÓA
      }
  }
  ```
- Với `preserveNullAndEmptyArrays = false`, nếu `ERA` bị null hoặc tìm không ra bên bảng `InsuranceERA`, toàn bộ Document đó sẽ bị drop (xóa) khỏi luồng Aggregation. 
- Do đó, mọi Document sống sót trôi xuống stage `$project` đều **chắc chắn 100% có object `$RemitPayment`**. Nếu thực tế record đó không có field `Type`, MongoDB Aggregate sẽ chỉ trả về Missing/Undefined tại field đó chứ không throw `NullReferenceException` hay `TypeError`.

**Ở tầng Client (`payerAnalyticsReportPTE.html`):**
- Angular 1.x Expression parser (dấu `{{ }}`) tự động có tính chất **Null-safe**. Nếu `remit.PaymentType` bị `undefined` hoặc `null`, AngularJS sẽ tự render thành Empty String (`""`) mà không gây lỗi throw undefined property.
- Việc anh chèn thẻ `<th ng-click="reorder(...)">` và xử lý sort data với field `PaymentType` hoàn toàn tuân thủ các component patterns hiện tại.
- Anh cũng đã bổ sung string localization: `ClinicReport.PaymentType` vào `translate.js` một cách rất chuẩn chỉnh.
