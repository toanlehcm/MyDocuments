### 1. Giải thích đoạn điều kiện HTML (`aria-expanded="..."`)
Đoạn code trong file `payerAnalyticsReportPTE.html` chính là UI Logic để **hiển thị icon mũi tên (lên/xuống)** khi người dùng tương tác sort cột:
*   Biến thứ nhất: `payerAnalyticsReportData.subSorting.sortType == 'PaymentType'`. Angular kiểm tra xem cái cột đang được user "focus" (click vào) có đúng là cột PaymentType không. Tránh tình trạng click vào cột `StatusCode` nhưng mũi tên lại hiện ở... `PaymentType`.
*   Biến thứ hai `payerAnalyticsReportData.subSorting.sortReverse`: Nó là biến Boolean lưu trữ **Chiều Sắp Xếp**.
    *   Nếu là **`true`** (có dấu `&&`): Ứng dụng render class HTML là `fa-angle-down` (mũi tên chỉ xuống - Sắp xếp giảm dần Z-A, 9-0).
    *   Nếu là **`false`** (có dấu `&& !...`): Ứng dụng render class HTML là `fa-angle-up` (mũi tên chỉ lên - Sắp xếp tăng dần A-Z, 0-9).

### 2. Giải thích hàm `_reorder(...)` trong `reportATP.js`
Đây là bộ não (Dispatcher) xử lý khi user click vào một thẻ `<th>`:
1.  **Chuyển đổi Trạng thái Sắp Xếp (Toggle Sorting State)**:
    *   Dòng `if (sorting.sortType == type)`: Nó check xem người dùng có click tiếp vào *cái cột đang được chọn sẵn* hay không. Nếu có (nghĩa là user click nhiều lần vào 1 cột `PaymentType`), nó sẽ lật ngược lại chiều sắp xếp (`sortReverse = !sortReverse`). Không đổi cột.
    *   `else { sorting.sortType = type; }`: Nếu user click sang một cột THỨ HAI (ví dụ từ cột *StatusCode* bấm sang *PaymentType*), nó sẽ chuyển `sortType` sang cột mới (chuỗi 'PaymentType').
2.  **Khối `if (isReloaded)` (Kích hoạt Server-side Fetch)**:
    *   Dự án PtE có 2 kiểu load data. Bảng Main (Group tổng) thì dùng Backend API Fetching nên lúc click nó sẽ `_reloadReport();`.
    *   Nhưng bảng con bên trong (`RemitDetails`) do dữ liệu đã lấy sẵn rồi, UI HTML gọi hàm như sau: `ng-click="reorder(..., 'PaymentType', false)"`. Vì truyền hardcode là `false`, nó không chui vào khối `isReloaded`, mà sẽ để dành cho Local Filter (pipe `orderBy`) tự động sắp xếp trên Browser (Client-side Sorting).
