Để tối ưu quy trình làm task Jira của anh, em đã chủ động viết lại **Root Cause** và sửa code luôn. Dưới đây là nội dung anh có thể copy vào ticket:

---

**Root Cause:**
The search payload sends the "ERA" payment type via `searchCriteria.searchKeyword`, but the server's search aggregation pipeline (`buildPayerAnalyticsPipeline` in `v2_ReportDAL.js`) fails to query this value. Specifically, the `$match` operator around line 6726 handles text-searching via the `$or` array for multiple fields (such as `PayerName`, `StatusCode`, etc.), but it inherently lacks the `PaymentType` field in the condition list array.

**Solution:**
- **Backend (`v2_ReportDAL.js`)**: Appended `{ 'PaymentType': searchRegex }` into the `$or` array inside `textSearchConditions` logic. Because `PaymentType` is already exposed in the preceding `$project` stage, this dynamically binds the string regex condition to accurately fetch and return correctly matching documents.
