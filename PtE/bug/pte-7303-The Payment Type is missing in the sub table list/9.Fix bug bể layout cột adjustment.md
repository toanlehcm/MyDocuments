# Fix: Subtable Layout Bug — Cột Adjustment Bị Lệch

## Tóm tắt vấn đề

Sau khi thêm cột **PaymentType** vào subtable (`thead-four`) của Payer Analytics Report, layout của cột **Adjustment** (và có thể cả **Payments**) bị vỡ — không thẳng hàng với header của maintable phía trên.

---

## 🔍 Root Cause Analysis

### Giai đoạn 1: Tìm chỗ xét width của subtable

**Subtable** (`thead-four`) trong `#payerAnalyticsReport` có width được kiểm soát ở 2 tầng:

#### Tầng 1 — CSS SCSS (`_reports.scss`, dòng 934–970)
```scss
// #payerAnalyticsReport > .content-table > #accordion
.accordion-content {
  .card-body {
    .thead-four {
      tr {
        td {
          width: 10%;  // ← ĐÂY là nơi duy nhất xét width cho các cell của subtable
        }
        th { ... }
      }
    }
  }

  @media only screen and (max-width: 991.98px) {
    width: 1024px;     // ← Fixed width cho mobile
    max-width: 1024px;
  }
}
```

> **Kết luận tầng 1:** `td { width: 10% }` áp dụng **đồng đều** cho **tất cả** các `<td>` trong subtable — không có class riêng per-column.

#### Tầng 2 — Main table CSS (`_reports.scss`, dòng 874–891)
```scss
// #payerAnalyticsReport > .content-table > .table > thead
.table thead tr {
  .w-th-1 { width: 16%; min-width: 165px; max-width: 165px; }
  .w-th-n { width: 10%; }
  .w-th-8 { width: 11%; }
  .w-th-9 { width: 11%; }
}
```

Main table có 9 cột với tổng width:
`16% + 10%×6 + 11% + 11% = 98%` → gần đủ 100%.

---

### ✅ Root Cause Confirmed (90% confidence)

> **Subtable có 11 cột nhưng CSS chỉ set `td { width: 10% }` đồng đều cho tất cả, tổng = 110% > 100%. Browser phải co cụm các cột cuối (Payments, Adjustments) để fit vào container → layout vỡ.**

Việc thêm cột `PaymentType` đã làm tăng số cột từ 10 lên 11 mà **không có bất kỳ điều chỉnh nào** về:
1. Width từng cột trong CSS
2. Fixed width của container khi responsive (`1024px`)
3. Width class riêng cho từng `<th>/<td>` trong subtable

## Proposed Changes

### Option A — **Recommended: Thêm width class riêng cho subtable (Clean Solution)**

Giống cách maintable dùng `.w-th-1`, `.w-th-n`, `.w-th-8`, `.w-th-9`, tạo width class riêng cho subtable.

#### [MODIFY] [_reports.scss](file:///d:/Sources/pteverywhere/Client/app/styles/PtE_Reports/_reports.scss)

Thay thế rule `td { width: 10% }` bằng các class có width cụ thể cho subtable:

```scss
// Trong #payerAnalyticsReport > .content-table > #accordion > .thead-four > tr
td {
  // xóa width: 10% chung
}
// Thêm class riêng:
.w-sub-id    { width: 9%; min-width: 90px; }   // Remit ID (narrow)
.w-sub-type  { width: 8%; min-width: 80px; }   // Payment Type (narrow - text ngắn)
.w-sub-n     { width: 8%; }                     // Status, Age
.w-sub-name  { width: 11%; min-width: 100px; } // Patient Name, Therapist
.w-sub-date  { width: 9%; min-width: 90px; }   // Date of Service, Payment Date
.w-sub-amt   { width: 9%; }                     // Charges, Payments, Adjustments
```

Tổng: `9 + 8 + 8 + 11 + 11 + 9 + 9 + 8 + 9 + 9 + 9 = ~100%` ✅

Đồng thời tăng fixed width responsive:
```scss
// Từ 1024px → 1150px (để chứa thêm 1 cột)
@media only screen and (max-width: 991.98px) {
  width: 1150px;      // ← +126px để chứa cột PaymentType
  max-width: 1150px;
}
```

#### [MODIFY] [payerAnalyticsReportPTE.html](file:///d:/Sources/pteverywhere/Client/app/views/report/payerAnalyticsReportPTE.html)

Thêm class vào từng `<th>` và `<td>` của subtable tương ứng.

---

### Option B — **Quick Fix: Dùng `min-width` cho subtable container**

Tăng fixed width container và để browser tự phân phối:

```scss
// _reports.scss dòng 961-968
@media only screen and (max-width: 991.98px) {
  width: 1200px;      // ← tăng từ 1024px
  max-width: 1200px;
}
```

Và đổi rule chung:
```scss
td { width: auto; min-width: 80px; }
```

**Trade-off B:**
- ✅ Ít code thay đổi
- ❌ Không kiểm soát được width từng cột → alignment với maintable vẫn không chính xác

---

### Option C — **UX Redesign: Bỏ 1 cột ít quan trọng**

Do subtable đã có **11 cột** trên không gian hẹp, xem xét:
- Bỏ cột **Age** (đã có trên maintable level)
- Hoặc merge **Date of Service** + **Payment Date** thành 1 cột toggle

**Trade-off C:**
- ✅ Gọn hơn về UX
- ❌ Mất thông tin, cần align với product decision
