Dạ, đây là một câu hỏi rất hay, đi sâu vào cơ chế cốt lõi của AngularJS. 

Để trả lời ngắn gọn nhất: **Cái giá trị `SearchStr` KHÔNG HỀ được gán lúc user click Search. Nó đã được gán LIÊN TỤC ngay trong lúc user đang gõ phím.**

Cơ chế này được gọi là **Two-way Data Binding** (Ràng buộc dữ liệu 2 chiều) thông qua `ng-model`. Cụ thể luồng gán value diễn ra như sau:

---

### Step 1: Cơ chế 2-Way Binding của `ng-model`
Khi anh nhìn vào thẻ input trong HTML:
```html
<input ng-model="payerAnalyticsReportData.SearchStr" ...>
```
Đoạn code này tạo ra một "sợi dây liên kết trực tiếp" giữa giao diện UI và cục data trong file controller `reportATP.js` mà anh vừa cung cấp.

Mỗi khi anh gõ một ký tự (ví dụ: gõ chữ `g`, rồi `e`, rồi `o`...), AngularJS sẽ tự động bắt sự kiện thay đổi (thông qua `$digest` cycle nội bộ) và cập nhật thẳng giá trị đó vào biến `$scope.payerAnalyticsReportData.SearchStr` ở phía dưới controller. 

*➜ Kết quả:* Ngay trước khi anh kịp nhấn nút Search, biến `SearchStr` đã chứa chuỗi "george" rồi, không cần bất kì hàm trung gian nào gán giá trị cả.

### Step 2: Hành động Click Search (Chỉ mang tính chất Trigger)
Khi anh click icon Search:
```html
<div ng-click="reportEnterSearch();">
```
Hàm `reportEnterSearch` được gọi. Hàm này **hoàn toàn không làm nhiệm vụ lấy dữ liệu từ ô input gán vào biến**. Nó chỉ làm đúng vai trò của một người ra lệnh: "User gõ xong rồi, lấy cục data hiện tại đem đi gọi API đi".

### Step 3: Lấy data đem đi đóng gói
Luồng sẽ chui xuống hàm `_getPayerAnalyticsRequetsParams` như ở câu hỏi trước anh đã vẽ diagram.
```javascript
function _getPayerAnalyticsRequetsParams(exportType) {
    var data = _getReportData(); // Gọi hàm này, nó trả về nguyên cục $scope.payerAnalyticsReportData
    
    var params = {
        searchCriteria: {
            ...
            "searchKeyword": data.SearchStr // Trích xuất giá trị đã được ng-model cập nhật sẵn
        },
        ...
    };
    return params;
}
```

---
**💡 TÓM LẠI:**
Lúc user ấn Enter hoặc Click icon tìm kiếm, hệ thống chỉ đơn giản là đi vào object `$scope.payerAnalyticsReportData`, đọc cái property `SearchStr` lên, nhét vào payload và gửi lên API. Giá trị của property đó đã được AngularJS "lén" gán (sync) vào liên tục trong quá trình user gõ phím thông qua sức mạnh của `ng-model` rồi anh nhé.
